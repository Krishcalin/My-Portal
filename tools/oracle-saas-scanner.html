01Overview

Oracle Fusion Cloud runs the financials, HR, supply chain, and customer data of thousands of enterprises, yet its security posture lives across two very different control planes: the IDCS / OCI IAM Identity Domain that governs authentication, and the Fusion, HCM, and EPM REST layers that govern roles, data security, and audit. Most SSPM platforms treat Oracle SaaS as a blind spot. This scanner closes it with 55 live security checks that authenticate to a real tenant and read policy directly from Oracle's own APIs.

Built as a single, dependency-light Python file, the scanner maps to the CIS Oracle Cloud Infrastructure SaaS Cloud Applications Benchmark v1.0.0 across four sections: Identity and Access Management, Configuration Management, Networking, and Logging and Monitoring. Every finding is enriched with mappings to NIST 800-53 Rev 5, ISO/IEC 27001:2022, and SOC 2 Type II, so a single run produces evidence usable by security engineering, GRC, and audit teams at once.

It is deliberately pragmatic about access. OAuth 2.0 Client Credentials is the primary auth path, with Basic Auth as a fallback, and it degrades gracefully: given only an IDCS URL it still runs the identity, network, and partial logging checks (32 of 55); add a Fusion URL and all 55 checks light up. Endpoints that return 403/404 log a warning and the scan continues, so a partially-permissioned service account still yields a useful report.

The audience is Oracle-heavy security teams, cloud posture engineers, and auditors who need a repeatable, scriptable read of Oracle SaaS hygiene. Output is a choice of ANSI console, machine-readable JSON, or a self-contained HTML dashboard, and a non-zero exit code on CRITICAL/HIGH findings makes it a drop-in CI/CD gate.

02Key Capabilities

55 live API-driven checks

Authenticates to a real Oracle tenant and evaluates 55 posture checks against live IDCS and Fusion Cloud REST responses rather than static config exports.

Four-section CIS coverage

Organizes checks into Identity and Access Management (22), Configuration Management (13), Networking (10), and Logging and Monitoring (10), mirroring the CIS Oracle SaaS benchmark structure.

Four compliance frameworks per finding

Every rule carries CIS Oracle SaaS v1.0.0, NIST 800-53 Rev 5, ISO 27001:2022, and SOC 2 Type II mappings sourced from a single 55-entry COMPLIANCE_MAP.

Dual authentication

Uses OAuth 2.0 Client Credentials as the primary path with HTTP Basic Auth as a fallback, plus automatic token refresh on expiry.

Dual API surface

Reads both the IDCS /admin/v1/ SCIM endpoints and the Fusion fscmRestApi/hcmRestApi/EPM interop REST endpoints in one pass.

Graceful degradation

Runs an IDCS-only mode (32 checks) when no Fusion URL is supplied and continues past per-endpoint 403/404 errors instead of aborting the scan.

Three output formats

Emits ANSI-colored console output, structured JSON, and a self-contained HTML dashboard with Oracle-red branding and client-side filtering.

CI/CD gate exit codes

Returns exit code 1 when CRITICAL or HIGH findings are detected and 2 on argument errors, enabling pipeline pass/fail policy.

Severity filtering

A --severity flag narrows the report to CRITICAL, HIGH, MEDIUM, or LOW so teams can focus remediation triage.

Zero-setup single file

The entire scanner is one Python file whose only third-party dependency is requests, making it trivial to drop into any environment.

Offline demo generator

run_demo.py injects synthetic findings to produce sample JSON and HTML reports with no live tenant or credentials required.

Rate-limit awareness

Handles HTTP 429 responses using the Retry-After header and paginates SCIM (startIndex/count) and Fusion (offset/limit) result sets.

03Architecture

A single-file scanner built around one OracleSaaSScanner class. Its scan() entry point authenticates once, then invokes 21 _check_* methods that pull policy from IDCS SCIM and Fusion REST endpoints, normalize each issue into a Finding object auto-enriched from COMPLIANCE_MAP, and finally serialize the collected findings through console, JSON, and HTML reporters.

1
Authentication layer
_authenticate() negotiates OAuth 2.0 Client Credentials against the IDCS token endpoint (falling back to Basic Auth) and refreshes the bearer token on expiry.
2
API access layer
_idcs_get/_idcs_get_single handle SCIM startIndex/count pagination while _fusion_get/_fusion_get_single handle offset/limit pagination, both with 403/404/429 handling.
3
Check engine
21 _check_* methods grouped into the four CIS sections evaluate live API data and emit findings via self._add(), covering all 55 rule IDs.
4
Finding + compliance model
A __slots__-based Finding class carries rule metadata; the 55-entry COMPLIANCE_MAP auto-populates CIS/NIST/ISO/SOC2 mappings for each rule ID.
5
Reporting layer
print_report() renders ANSI console output, save_json() writes structured JSON, and save_html() produces a self-contained Oracle-branded HTML dashboard.
6
CLI orchestration
main() parses argparse flags/env vars, wires up the scanner, applies severity filtering, and sets exit codes for CI/CD gating.

05Security Controls

Password policy (IAM-001..009)
Checks minimum length, upper/lower/numeric/special character requirements, expiration window, history depth, and account lockout threshold/duration against IDCS PasswordPolicies.
Multi-factor authentication (IAM-010..012, 022)
Flags MFA not enforced for all users or admins (CRITICAL), weak-only factors like SMS/Email, and Sign-On Policy MFA bypass rules (CRITICAL).
SSO and OAuth (IAM-013..015)
Detects missing SAML/SSO federation, OAuth clients with excessive admin scopes, and stale OAuth clients unmodified for 180+ days.
Privileged access and lifecycle (IAM-016..019)
Identifies over-provisioned IDCS privileged groups and Fusion privileged roles, users inactive 90+ days, and terminated users still active.
Admin access restrictions (IAM-020..021)
Checks for missing IP-based restriction on admin Sign-On Policies and self-registration enabled in the Identity Domain.
Configuration management (CFG-001..013)
Covers change-monitoring, unused/admin custom roles, approval workflows, sandbox promotion, flexfield security, unrestricted data roles, view-all security profiles, EPM snapshots, and privileged integration/scheduled accounts.
Network security (NET-001..010)
Evaluates Location-Based Access Control, broad CIDR perimeters, WAF coverage, wildcard CORS, IP filtering, token-endpoint exposure, session timeout, persistent sessions, and RFC1918 ranges in perimeters.
Logging and monitoring (LOG-001..010)
Checks Fusion/EPM/IDCS audit export, Separation-of-Duties violation monitoring (CRITICAL), retention period, real-time privileged-op alerting, login-failure monitoring, audit trail, SIEM integration, and diagnostic log level.
Severity model
Findings are graded CRITICAL/HIGH/MEDIUM/LOW, with CRITICAL reserved for MFA bypass and SoD gaps, driving both triage filtering and CI exit codes.

07Quick Start

$ pip install requests
$ python oracle_saas_scanner.py --idcs-url https://idcs-abc123.identity.oraclecloud.com --client-id <client-id> --client-secret <client-secret>
$ python oracle_saas_scanner.py --idcs-url https://idcs-abc123.identity.oraclecloud.com --fusion-url https://myco.fa.us2.oraclecloud.com --client-id <id> --client-secret <secret> --html report.html --json report.json
$ python oracle_saas_scanner.py --idcs-url https://idcs-abc123.identity.oraclecloud.com --client-id <id> --client-secret <secret> --severity HIGH
$ python oracle_saas_scanner.py --idcs-url https://idcs-abc123.identity.oraclecloud.com --username admin@myco.com --password <password>
$ python run_demo.py   # generate sample JSON + HTML reports offline
