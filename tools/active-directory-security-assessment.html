01Overview

Active Directory Security Assessment is a read-only, Python-based auditor for Microsoft AD environments. It connects over LDAP/LDAPS (and optionally WinRM) and runs 86 defensive checks across 15 categories - Kerberos attack surface, privilege escalation, AD CS certificate misconfigurations, password policy, trust relationships, authentication weaknesses, and more - then produces an interactive HTML report and machine-readable JSON with a 0-100 posture score and letter grade.

The design philosophy is deliberately non-invasive: every AD query is an LDAP search operation and every remote command is a PowerShell Get-* cmdlet, so the scanner never modifies a single object. That makes it safe to run against production domain controllers by a Domain Users account (Domain Admins unlocks full coverage), and safe to wire into CI/CD - the process exits 0 when clean, 1 on HIGH findings, and 2 on CRITICAL.

Coverage is split across two data sources. Roughly 70% of checks are answered directly from LDAP/LDAPS (Kerberoasting and AS-REP roasting candidates, delegation abuse, krbtgt age, AD CS ESC templates, privileged-group membership, trusts), while about 30% require an optional WinRM session for host-side registry and audit-policy settings (WDigest, LM hash storage, SMB signing, Credential Guard, LAPS, NTLM restriction). WinRM-dependent checks degrade gracefully to INFO notifications when the session is unavailable, so an LDAP-only run still returns actionable results.

It is aimed at blue teams, AD administrators, and internal auditors who need a repeatable, evidence-backed snapshot of domain security posture. Every finding ships with a severity, the observed-versus-expected state, a plain-English recommendation, and a copy-paste PowerShell remediation command (display only, never executed), turning a scan into a prioritized, hardening to-do list.

02Key Capabilities

86 read-only AD security checks

Audits 86 distinct misconfigurations across 15 attack-relevant categories using only LDAP search and PowerShell Get-* commands, never mutating the directory.

Kerberos attack-surface analysis

Flags Kerberoastable and AS-REP-roastable accounts, unconstrained/constrained/RBCD delegation, DES-only encryption, stale krbtgt passwords, and an empty Protected Users group.

AD CS (ESC) certificate abuse detection

Inspects certificate templates and enrollment services for ESC1 (SAN + client auth), ESC2 (Any Purpose EKU), ESC6 (EDITF_ATTRIBUTESUBJECTALTNAME2), and ESC8 (HTTP enrollment/NTLM relay).

Privilege-escalation and DCSync mapping

Enumerates Domain/Enterprise/Schema Admin membership, stale and orphaned adminCount accounts, dangerous ACLs, AdminSDHolder tampering, and non-standard principals holding DCSync replication rights.

Posture scoring with letter grades

Distills all findings into a single 0-100 score via severity-weighted penalties (CRIT x15, HIGH x5, MED x2, LOW x0.5) mapped to an A-F grade for at-a-glance reporting.

Self-contained interactive HTML report

Emits a single dark-theme HTML file with an animated score ring, severity cards, and live client-side search plus severity/category filters - no external assets or internet required.

Dual LDAP + WinRM data collection

Combines directory-level LDAP/LDAPS evidence with optional host-side WinRM registry and audit-policy checks, gracefully skipping WinRM checks with INFO notices when unavailable.

PowerShell remediation for every finding

Each finding carries a ready-to-run (display-only) remediation command such as Set-ADAccountControl or Reset-ADServiceAccountPassword, so operators know exactly how to fix it.

CI/CD-friendly exit codes

Returns exit 0 (clean), 1 (HIGH findings), or 2 (CRITICAL findings) so scans can gate pipelines or trigger alerts automatically.

Selective scans and YAML profiles

Run all 15 categories or a comma-separated subset via --checks, or drive enablement from a reusable default_profile.yaml scan profile.

Flexible authentication and transport

Supports simple or NTLM binds, plaintext LDAP (389) or encrypted LDAPS (636), optional SSL verification skip, and configurable ports/timeouts.

Offline-testable check modules

Each checker is independently unit-tested against mock LDAP responses, so 28 tests validate detection logic with no live domain controller.

03Architecture

A single CLI entry point orchestrates a linear connect -> discover -> assess -> score -> report pipeline. The scanner authenticates a read-only LDAP/LDAPS client and an optional WinRM client, then iterates a registry of 15 pluggable checker classes (ALL_CHECKERS) that each return a list of finding dicts; findings are aggregated, scored by a severity-weighted posture model, and rendered to self-contained HTML and JSON.

1
CLI orchestrator (ad_security_scanner.py)
Argparse-driven entry point that loads the profile, resolves enabled categories, wires up clients, runs each checker, prints the console summary, and sets severity-based exit codes.
2
LDAP client (utils/ldap_client.py)
Read-only ldap3 wrapper that auto-discovers the base DN from RootDSE and exposes typed getters (get_users, get_groups, get_trusts, get_certificate_templates, get_fine_grained_password_policies, etc.).
3
WinRM client (utils/winrm_client.py)
Optional pywinrm client running only Get-* PowerShell for host-side settings (audit policy, SMB config, WDigest, LM hash, Credential Guard, LAPS, NTLM) with graceful connect-failure fallback.
4
Check modules (checks/*.py)
15 independent checker classes registered in ALL_CHECKERS; each run_all() invokes private _check_ methods that emit structured findings with id, severity, actual/expected, recommendation, and remediation_cmd.
5
Severity & scoring engine (utils/severity.py)
Defines severity ordering and weights, computes the clamped 0-100 posture score, converts it to an A-F grade, and tallies severity counts.
6
Report generator (utils/report_generator.py)
Renders findings into a self-contained dark-theme HTML report (score ring, severity cards, live search/filter JS) and a structured JSON report with metadata and severity summary.

05Security Controls

Kerberos Security
Detects Kerberoastable users and admins (SPN-bearing accounts), AS-REP roasting (DONT_REQUIRE_PREAUTH), unconstrained/constrained/RBCD delegation, DES-only encryption, krbtgt passwords older than 180 days, and empty Protected Users group.
Certificate Services (AD CS)
Audits certificate templates and enrollment services for ESC1 (enrollee-supplied SAN + client auth), ESC2 (Any Purpose EKU), ESC6 (EDITF_ATTRIBUTESUBJECTALTNAME2 on the CA), and ESC8 (HTTP/NTLM-relay enrollment).
Privileged Accounts
Reviews Domain/Enterprise/Schema Admin and built-in Administrator membership, nested and dangerous groups, stale admins, orphaned adminCount, and Kerberoastable privileged accounts.
Password Policy
Eleven checks covering minimum length, complexity, lockout threshold, history, max age, reversible encryption, fine-grained password policies, and password-never-expires/stale-password accounts (min-length advice aligned to CIS 14+).
Authentication Security
Flags NTLMv1/LM hash storage, WDigest credential caching, missing SMB signing, SMBv1 on DCs, and unrestricted/unaudited NTLM - primarily via WinRM registry inspection.
Replication & Object Permissions
Identifies non-standard principals with DCSync rights, DFSR/FRS SYSVOL replication and health issues, Pre-Windows 2000 Compatible Access membership, dangerous domain-object ACLs, and AdminSDHolder tampering.
LDAP & DNS Security
Checks LDAP signing, channel binding, anonymous bind and LDAPS enforcement, plus DNS zone transfers, insecure dynamic updates, scavenging, and WPAD block-list hardening.
Trusts & Domain Configuration
Evaluates SID filtering, selective authentication and bidirectional trusts, alongside domain functional level, AD Recycle Bin, machine account quota, DC OS versions, and stale DCs.
Group Policy, Lateral Movement & Auditing
Covers audit policy, PowerShell/command-line logging, Credential Guard, LAPS deployment, RDP/Restricted Admin and Remote Credential Guard restrictions, event-log sizing, and unlinked GPOs.
Read-only safety model
All directory access uses LDAP search operations and all remote execution uses PowerShell Get-* cmdlets only - no Set-/New-/Remove-, so scans never alter AD; remediation commands are surfaced for the operator but never run.

07Quick Start

$ pip install -r requirements.txt
$ python ad_security_scanner.py --server dc01.corp.com --domain corp.com --user scanner@corp.com --password MyP@ss --html report.html --json report.json
$ python ad_security_scanner.py --server dc01.corp.com --domain corp.com --user scanner@corp.com --password MyP@ss --ssl
$ python ad_security_scanner.py --server 10.0.0.1 --domain corp.com --user CORP\scanner --password MyP@ss --auth ntlm
$ python ad_security_scanner.py --server dc01.corp.com --domain corp.com --user scanner@corp.com --password MyP@ss --no-winrm --checks kerberos_security,password_policy
$ python -m unittest tests.test_scanner -v

08Compliance & Frameworks

CIS Microsoft AD Benchmark
Cited as a hardening reference in remediation guidance (e.g. minimum password length 14+) and in the project's design goals.
MITRE ATT&CK
Checks correspond to recognized enterprise techniques such as Kerberoasting (T1558.003), AS-REP roasting (T1558.004), and DCSync (T1003.006).
AD CS ESC taxonomy
Implements the SpecterOps AD CS escalation classifications ESC1, ESC2, ESC6, and ESC8 for certificate-service misconfiguration detection.
ANSSI AD Security Guide
Referenced as a design guideline for privileged-access and delegation hardening checks.
