01Overview

Most cloud security tooling treats Oracle Cloud Infrastructure as an afterthought, and the CNAPP products that do cover it are heavyweight SaaS agents that demand live API credentials into your tenancy. OCI CNAPP Security Scanner takes the opposite approach: it is a single, self-contained Python tool that assesses an entire OCI estate entirely offline, by reading the JSON that the standard `oci` CLI already produces. Nothing ever connects back to your cloud, which makes it safe to run in air-gapped review environments, on an auditor's laptop, or inside a CI pipeline.

The scanner implements all five pillars Gartner uses to define a Cloud-Native Application Protection Platform in one place: CSPM (posture and misconfiguration), CIEM (IAM entitlements and least privilege), CWPP (compute, containers, bastion, WAF), KSPM (OKE/Kubernetes posture), and IaC scanning of Terraform state and plans. That breadth is delivered through 96 discrete checks organized into 13 auditor modules, mapped to the CIS Oracle Cloud Infrastructure Foundations Benchmark v3.1.0 (~95% coverage) plus OCI best practices and industry frameworks such as NIST 800-53, ISO 27001, and SOC 2.

It is deliberately dependency-free (Python 3.8+ standard library only) and clone-and-run: there is nothing to install, pin, or keep patched. Every scan produces an interactive HTML dashboard with a weighted risk score, severity breakdown, and per-finding remediation guidance, and falls back to structured JSON when the report layer is unavailable. Module selection and severity filtering make it equally usable for a one-off audit or a repeatable check gate.

It suits cloud security engineers, OCI platform teams, and auditors who need a fast, defensible read on tenancy posture without granting a third-party product standing access to production. A bundled sample_data set of deliberately misconfigured exports means you can see a full report in seconds, then point it at real `oci` CLI output when ready.

02Key Capabilities

Unified five-pillar CNAPP

Covers CSPM, CIEM, CWPP, KSPM, and IaC security in a single scanner instead of five separate tools.

96 checks across 13 modules

Purpose-built auditors span IAM, networking, compute, storage, database, logging, Cloud Guard, OKE, Vault/KMS, WAF, bastion, Terraform, and CIS compliance.

Fully offline analysis

Reads local `oci` CLI JSON exports and Terraform state/plans and never connects to a live tenancy, making it safe for isolated review environments.

Zero external dependencies

Runs on the Python 3.8+ standard library alone, so there is nothing to install or keep patched.

CIS OCI Foundations v3.1.0 mapping

Checks carry CIS section references with roughly 95% coverage of the benchmark for defensible, framework-aligned reporting.

Interactive HTML dashboard

Generates a self-contained report with a weighted risk score, severity and category breakdowns, and detailed affected-item lists.

Selective module execution

Run all modules or target a subset (e.g. `--modules iam network compute oke`) to focus an assessment or speed up reruns.

Severity filtering

The `--severity` flag limits output to CRITICAL/HIGH/MEDIUM/LOW so pipelines and reviewers can triage the findings that matter.

Terraform IaC scanning

Inspects `terraform.tfstate` and plan JSON for local-state storage, risky resource deletions, and insecure state configuration.

Flexible data loading

A FILE_MAP with alternate filenames and an OCI CLI `{"data": [...]}` envelope unwrapper ingest real exports with minimal massaging.

Baseline threshold overrides

An optional `--config` JSON lets teams tune thresholds like max admin-group size or API-key age to their own standards.

Ready-to-run sample estate

Bundled deliberately misconfigured JSON exports produce a complete demonstration report out of the box.

03Architecture

A thin CLI orchestrator (oci_scanner.py) loads all OCI exports once through a DataLoader, then dispatches a MODULE_MAP of 13 auditor classes that each inherit a shared BaseAuditor contract; every auditor emits normalized finding dicts that are severity-filtered and rendered by a standalone HTML ReportGenerator (with a JSON fallback). Data flows one way: JSON exports to loaded dict to per-module checks to findings to dashboard.

1
CLI orchestrator (oci_scanner.py)
Parses arguments, holds the MODULE_MAP of 13 modules, runs the selected auditors, applies severity filtering, and drives report generation.
2
DataLoader (modules/base.py)
Iterates FILE_MAP to load every OCI CLI JSON export (with alternate filenames and BOM-tolerant reads) into a single in-memory dict.
3
BaseAuditor contract (modules/base.py)
Shared base class providing the finding() schema, severity constants, baseline lookups, and the _unwrap() helper that strips the OCI CLI data envelope.
4
CSPM core auditors (modules/cspm_core.py)
IamPolicyAuditor, NetworkVcnAuditor, ComputeAuditor, and StorageAuditor covering IAM/CIEM, VCN, compute, and storage posture.
5
CIEM & telemetry auditors (modules/cspm_ciem.py)
DatabaseAuditor, LoggingAuditAuditor, and CloudGuardAuditor covering database security, logging/audit/monitoring, and Cloud Guard detection.
6
CWPP/KSPM/IaC auditors (modules/cwpp_iac.py)
OkeSecurityAuditor, VaultKmsAuditor, WafEdgeAuditor, BastionAuditor, IacSecurityAuditor, and CisBenchmarkAuditor for workload, encryption, edge, and IaC/compliance checks.
7
ReportGenerator (modules/report_generator.py)
Renders findings into a self-contained interactive HTML dashboard with a weighted risk score and severity/category breakdowns, falling back to JSON.

05Security Controls

IAM & Entitlements (CIEM) - 20 checks
Root-compartment resources, MFA enforcement, overly broad and tenancy-wide policies, API-key and auth-token/secret-key rotation, admin-group size and protection, unused credentials, multiple/admin API keys, federation, and password policy (expiry/reuse).
Networking & VCN - 12 checks
Default security-list egress, SSH/RDP open to 0.0.0.0/0 (security lists and NSGs), overly permissive NSG rules, missing VCN flow logs, internet/service gateways, route-table issues, and unrestricted ICMP.
Compute hardening - 8 checks
Shielded Instance Secure Boot and Measured Boot, in-transit encryption, public IPs, legacy images, boot-volume customer-managed keys, IMDSv1 usage, and monitoring agent enablement.
Storage exposure & encryption - 8 checks
Public buckets, versioning, lifecycle and replication policies, customer-managed key usage, pre-authenticated request review, plus block-volume and file-storage CMK checks.
Database security - 6 checks
Autonomous DB CMEK and IAM authentication, DB Systems automated backups, MySQL HeatWave private endpoints, private-subnet placement, and patch currency.
Logging, Audit & Monitoring - 12 checks
Audit-log retention (365 days), VCN flow logs, service connectors, events rules and alarms, log-group organization, default tags, notification topics, IAM/VCN event rules, Cloud Guard and object-storage write logging, and local-user auth notifications.
Cloud Guard threat detection - 5 checks
Cloud Guard enablement, detector and responder recipes, security zones, and unresolved Cloud Guard problems.
OKE / Kubernetes (KSPM) - 8 checks
Public API endpoints, network-policy and pod-security enforcement, RBAC-to-OCI-IAM integration, node secure boot, workload identity, image validation, and node auto-upgrade.
Vault & KMS encryption - 5 checks
Key rotation, customer-managed key usage, vault replication, overly broad key-access policies, and HSM protection level.
Edge, Bastion & IaC - 12 checks
WAF policy/rules/rate-limiting/custom-rule coverage on load balancers, Bastion service usage/session limits/audit logging and direct-public-SSH detection, Terraform state/plan misconfiguration and local-state checks, and a CIS benchmark summary.

07Quick Start

$ git clone https://github.com/Krishcalin/OCP-CNAPP-Security-Scanner.git && cd OCP-CNAPP-Security-Scanner
$ python oci_scanner.py --data-dir ./sample_data --output report.html   # full scan of bundled sample estate
$ python oci_scanner.py --data-dir ./exports --severity HIGH            # only CRITICAL/HIGH findings
$ python oci_scanner.py --data-dir ./exports --modules iam network compute oke   # targeted modules
$ oci iam user list --compartment-id $TENANCY_OCID --all > exports/iam_users.json   # collect real data via OCI CLI
$ python oci_scanner.py --data-dir ./exports --config baseline.json     # apply custom thresholds
