01Overview

AWS Security Scanner is a two-in-one AWS security toolkit that covers both sides of the cloud lifecycle from a single repository: catch misconfigurations in infrastructure code before it ships, and audit what is actually running in a live account afterwards. Both tools are pure Python with a deliberately small footprint, so they drop into a laptop, a CI runner, or a compliance workstation with almost no setup.

The IaC Security Scanner (aws_offline_scanner.py) performs pure static analysis of Terraform (.tf) and CloudFormation (YAML/JSON) files with no AWS credentials and no network access. It combines 59 Terraform regex SAST rules with 42 CloudFormation resource-type structural checks across 25+ AWS services, emitting console, JSON, or interactive HTML reports and a non-zero exit code on CRITICAL/HIGH findings so it can gate a pipeline. Because it never touches AWS, security teams can review a vendor's or a developer's templates before anything is provisioned.

The Live Audit Scanner (aws_live_scanner.py) connects to a running account through boto3 using only read-only describe/get/list calls, executing 98 distinct checks across 25 service sections. Every finding is auto-tagged with severity, mapped controls from five compliance frameworks (CIS AWS v3.0, PCI DSS v4.0, HIPAA, SOC 2, NIST 800-53 Rev 5), and a copy-paste AWS CLI remediation command. It rolls results into a 0-100 posture score with an A-F letter grade and saves CSV/JSON evidence artefacts for auditors.

Together the two scanners give a security leader a shift-left gate and a runtime assurance check that speak the same language of AWS services, severities, and compliance controls. The live scanner ships with 28 mock-boto3 unit tests so its check logic can be validated without ever hitting a real AWS account.

02Key Capabilities

Dual-mode coverage (pre-deploy and runtime)

One repo pairs a credential-free IaC static analyzer with a live boto3 account auditor so misconfigurations are caught both before and after deployment.

Terraform SAST engine

59 regex-based rules (TF_SAST_RULES) scan .tf files line-by-line, skipping comments, to flag insecure resource arguments across 25+ services.

CloudFormation structural analysis

A CF_DISPATCH table routes 42 AWS:: resource types to dedicated _cf_* check methods that inspect template structure rather than raw text.

Custom CloudFormation YAML loader

A purpose-built loader (_CF_LOADER) parses CloudFormation intrinsic tags like !Ref, !Sub, and !GetAtt that would otherwise break a standard YAML parser.

Read-only live account audit

The live scanner uses only describe/get/list boto3 calls and never modifies AWS resources, so it is safe to run against production.

Five-framework compliance mapping

Each finding is tagged via COMPLIANCE_MAP with the applicable CIS, PCI DSS, HIPAA, SOC 2, and NIST 800-53 controls.

Posture scoring with letter grade

compute_risk_score turns findings into a severity-weighted 0-100 score and score_to_grade converts it to an A-F grade for executive reporting.

Actionable AWS CLI remediation

REMEDIATION_MAP attaches a ready-to-run AWS CLI command to every failed check so engineers can fix issues without hunting through docs.

Evidence artefact collection

save_evidence writes per-check CSV/JSON artefacts to a timestamped output directory to support audit and compliance workflows.

Three output formats per scanner

Both tools produce colour-coded console output, machine-readable JSON, and a self-contained dark-themed interactive HTML report with severity and search filtering.

Section-scoped and severity-filtered scans

The live scanner can run only named sections (e.g. IAM S3 VPC) and the IaC scanner can suppress findings below a chosen severity threshold.

CI/CD-friendly exit codes

The IaC scanner returns exit code 1 when any CRITICAL or HIGH finding is present, making it a drop-in pipeline gate for infrastructure code.

03Architecture

Two independent single-file Python scanners share a common design language of Findings/Results, severities, and compliance tags. The IaC scanner runs a Terraform regex-SAST pass and a CloudFormation structural pass over local files; the live scanner drives lazy-cached boto3 clients through 25 section methods, then enriches every result with severity, compliance, and remediation before scoring and reporting.

1
IaC input dispatch
AWSIaCScanner.scan_path walks a file or directory and _dispatch_file routes .tf to the Terraform engine and .yaml/.yml/.json to the CloudFormation engine, skipping package/tsconfig JSON.
2
Terraform SAST engine
_sast_scan applies the 59 TF_SAST_RULES regex patterns line-by-line, producing Finding dataclasses (rule_id, severity, file/line, description, recommendation, CWE).
3
CloudFormation structural engine
A custom _CF_LOADER parses intrinsic tags, then CF_DISPATCH maps 42 AWS:: resource types via getattr to 39 dedicated _cf_* methods that evaluate resource properties.
4
Live boto3 audit engine
AWSLiveScanner._client lazily caches boto3 clients and a CHECK_MAP of 25 sections invokes read-only check methods that call self._add to record Results.
5
Enrichment and scoring
self._add auto-populates severity (CHECK_SEVERITY), compliance controls (COMPLIANCE_MAP), and remediation (REMEDIATION_MAP); compute_risk_score and score_to_grade yield the 0-100 posture score and A-F grade.
6
Reporting and evidence
Both scanners emit colour console, JSON, and interactive HTML reports; the live scanner additionally writes CSV/JSON evidence artefacts to an output directory.

05Security Controls

IAM & identity (IaC + live)
Wildcard actions/principals, AdministratorAccess attachment, inline user policies, password reset; live checks for root MFA + access keys, console users without MFA, password policy, stale keys, and IAM Access Analyzer.
S3 exposure
Public-read/public-read-write ACLs, all four Block Public Access flags, versioning, encryption, and logging; live account-level BPA plus per-bucket public access, ACLs, and encryption.
Network & VPC
Security groups exposing SSH/RDP/0-65535 to 0.0.0.0/0, VPC Flow Logs, and public-subnet auto-assign public IP settings.
EC2 & EBS
IMDSv2 enforcement (no IMDSv1), public IP assignment, and EBS/volume encryption at rest.
Encryption & key management
KMS key rotation, SQS/SNS/Secrets Manager KMS encryption, DynamoDB CMK/SSE, ElastiCache at-rest and in-transit encryption plus AUTH tokens, and RDS storage encryption.
Logging & threat detection
CloudTrail multi-region + log-file validation + encryption, AWS Config recorders, GuardDuty detectors, Security Hub, CloudWatch log retention/KMS, and Route 53 query logging/DNSSEC.
Data services (RDS, Redshift, OpenSearch)
Public accessibility, encryption, automated backups, deletion protection, Multi-AZ, enhanced monitoring, public snapshots, HTTPS enforcement, node-to-node encryption, and VPC deployment.
Compute & containers (Lambda, EKS, ECS, ECR)
Public function access, VPC config, plaintext secrets in env vars, deprecated runtimes, EKS public API endpoint and control-plane logging, ECS privileged/root/writable-rootfs containers, and ECR image scan-on-push and tag immutability.
Edge & app protection (CloudFront, ELB, WAF, API Gateway)
HTTPS-only/TLS-version enforcement, WAF Web ACL presence/default-action/logging, ELB HTTPS listeners, and API Gateway stage logging.
AI/ML (Bedrock, Bedrock Agents, SageMaker)
Bedrock model-invocation logging, guardrails, KMS encryption and VPC endpoints; agent execution-role and knowledge-base security with prompt-injection considerations; SageMaker internet access and storage encryption.

07Quick Start

$ python aws_offline_scanner.py /path/to/infra/  # scan a directory of Terraform + CloudFormation files
$ python aws_offline_scanner.py main.tf --json findings.json --severity HIGH  # IaC scan, JSON report, HIGH+ only
$ python aws_offline_scanner.py template.yaml --html report.html  # IaC scan to interactive HTML
$ python aws_live_scanner.py  # full live audit of all 25 sections (default region eu-west-1)
$ python aws_live_scanner.py --region us-east-1 --sections IAM S3 VPC  # scoped live audit of specific sections
$ python aws_live_scanner.py --json report.json --html report.html --output-dir ./audit_output  # reports + evidence artefacts
