01Overview

Static analysis finds bugs in code that never runs the way production does. DAST Scanner attacks the gap: it points a real HTTP client at a live web application, crawls it like a user, injects genuine attack payloads, and reads the responses for proof of exploitation. It is a single, portable Python file with exactly one third-party dependency (requests) — the entire crawler, attack engine, WAF detector, authenticator, and four-format reporter fit in ~3,650 lines of Python 3.10+ that runs unchanged on Windows, macOS, and Linux.

Under the hood it ships 109 distinct security checks organised into 23 vulnerability categories and mapped to the OWASP Top 10 2021 and CWE. Coverage spans the full active-testing surface: error/time/boolean-blind SQL injection, reflected and DOM XSS, SSRF and open redirect, LFI/RFI, XXE, JWT algorithm and signature flaws, insecure deserialization across four runtimes, file-upload bypasses, HTTP request smuggling, web cache poisoning, OAuth and WebSocket weaknesses, plus a curated pack of 10 named CVE probes (Log4Shell, Spring4Shell, Struts, Text4Shell, MOVEit, and more).

Built for real engagements and pipelines, it detects eight major WAF vendors before it starts, enforces strict single-host scope so it never wanders off-target, throttles itself with a token-bucket rate limiter, and supports five authentication modes (none, bearer, cookie, basic, form-login) for testing behind a login. Output comes as coloured console, JSON, an interactive HTML report, and SARIF for GitHub/GitLab code scanning — and it returns exit code 1 on CRITICAL/HIGH findings so it fails a CI build cleanly.

It is aimed at penetration testers, application-security engineers, and DevSecOps teams who want a dependency-light, auditable DAST tool they can drop into a repo, route through Burp or ZAP, and read end to end. A bundled intentionally-vulnerable app (stdlib only) lets anyone validate the scanner in two terminals without touching a production target.

02Key Capabilities

109 active security checks

Every check injects a real payload or probe and confirms exploitation from the response, spanning 23 vulnerability categories mapped to OWASP Top 10 2021 and CWE.

BFS web crawler with pre-crawl seeding

Breadth-first crawl with configurable depth and page caps, seeded from robots.txt and sitemap.xml, extracting URLs, HTML forms, and API endpoints.

WAF detection for 8 vendors

Fingerprints Cloudflare, AWS WAF, Imperva, Akamai, ModSecurity, F5 BIG-IP, Sucuri, and Barracuda from headers/cookies/body before testing begins.

Five authentication modes

Scans behind a login using bearer tokens, session cookies, HTTP basic auth, or automated form-based login for authenticated attack surface.

Technology fingerprinting

Identifies server, framework, CMS, and language from response signals to focus checks and enrich findings.

Token-bucket rate limiting and scope lock

Throttles to a configurable requests-per-second budget and enforces single-host scope so the scanner never leaves the target.

Four report formats

Emits coloured console output, machine-readable JSON, a self-contained interactive HTML report, and SARIF for CI code-scanning dashboards.

Parallel check execution

Runs the 22 check modules concurrently across a 4-worker ThreadPoolExecutor to keep scans fast.

Proxy passthrough for Burp/ZAP

Routes all traffic through any HTTP proxy so testers can intercept, replay, and audit every request the scanner sends.

CI-friendly exit codes

Returns exit code 1 when CRITICAL or HIGH findings exist and 0 otherwise, so a pipeline gate fails deterministically.

Single-file, one-dependency design

The whole scanner is one Python file needing only the requests library, making it trivial to vendor into any repo or CI runner.

Bundled vulnerable test app

A stdlib-only intentionally-vulnerable web app ships in tests/ so users can validate detections safely and offline.

03Architecture

A single-file Python scanner organised as a five-stage pipeline over a shared HTTP client. The target URL flows through authentication, WAF detection, and BFS crawling to build a SiteMap (URLs, forms, API endpoints, tech), which 22 check modules consume in parallel to emit typed Finding objects that the reporter renders to console, JSON, HTML, and SARIF.

1
HTTP client & rate limiter
HTTPClient wraps requests with a token-bucket RateLimiter, request budget, timeout, optional TLS verify, proxy routing, and allowed-hosts scope enforcement.
2
Auth & WAF pre-flight
AuthManager establishes a session across five modes and detect_waf fingerprints eight WAF vendors before any attack traffic is sent.
3
Crawler & fingerprinting
WebCrawler runs a BFS crawl seeded by robots.txt/sitemap.xml, using _LinkFormParser (stdlib html.parser) to extract links and forms and detect server/framework/CMS/language into a SiteMap.
4
Parallel check engine
The CHECK_MODULES registry of 22 generator-based modules runs on a 4-worker ThreadPoolExecutor, yielding Finding dataclasses with rule_id, severity, evidence, CWE, and OWASP mapping.
5
Multi-format reporter
The report layer serialises findings via save_json, save_html, and save_sarif plus coloured console output, and drives the CRITICAL/HIGH exit-code gate.

05Security Controls

Injection (7 rules)
Error-based, time-based blind, and boolean-blind SQL injection via URL params and forms, plus OS command injection and NoSQL injection.
Cross-Site Scripting (5 rules)
Reflected XSS via URL params, forms, and HTTP headers; DOM sink detection; and reflected content in error pages.
SSRF & open redirect (3 rules)
Server-side request forgery via URL parameter and form input, plus open-redirect detection.
File inclusion & XXE (5 rules)
Local/Remote File Inclusion and backup-file exposure, plus XXE external-entity file disclosure and XXE via SOAP endpoints.
Authentication & session (7 rules)
Missing CSRF tokens, HttpOnly/Secure/SameSite cookie flags, default-credential acceptance, username enumeration, and session fixation.
Information disclosure (11 rules)
Exposed .env, .git, wp-config.php, DB connection strings, API keys, stack traces, phpinfo, internal IPs, and server-version leakage.
Security headers & CORS (9 rules)
Missing CSP, X-Content-Type-Options, X-Frame-Options, HSTS, Referrer-Policy, Permissions-Policy, plus wildcard and reflected CORS origins.
JWT, deserialization & upload (11 rules)
JWT alg=none / unverified signature / weak secret; Java/.NET/PHP/Python deserialization; and unrestricted upload with extension/content-type bypass.
Modern web attacks (16 rules)
HTTP request smuggling (CL.TE/TE.CL), web cache poisoning/deception, OAuth flow flaws, WebSocket origin/transport issues, and host-header injection.
Named CVEs, CMS & SSL/TLS (23 rules)
Probes for Log4Shell, Spring4Shell, Struts, Text4Shell, MOVEit and more; WordPress/Joomla/Drupal weaknesses; and expired/weak-TLS/mixed-content checks.

06Technology Stack

Language
Python 3.10+ (uses modern typing, dataclasses, from __future__ annotations)
Dependencies
Single external dependency: requests; everything else is Python stdlib
Concurrency
concurrent.futures.ThreadPoolExecutor (4 workers) for parallel check modules
Parsing
stdlib html.parser for link/form extraction; re, json, base64, hmac, hashlib for probes
Packaging
Single-file script (dast_scanner.py); no build step, vendorable into any repo
Testing
Bundled stdlib-only vulnerable app (tests/vulnerable_app.py) for integration validation; run_demo.py for offline mock reports
Deployment
Runs anywhere Python 3.10+ is available (Windows/macOS/Linux); proxy support for Burp/ZAP

07Quick Start

$ pip install requests  # only external dependency
$ python dast_scanner.py https://example.com --verbose --html report.html --json report.json
$ python dast_scanner.py https://api.example.com --auth-mode bearer --auth-token "your-jwt-token"
$ python dast_scanner.py https://example.com --proxy http://127.0.0.1:8080  # route through Burp/ZAP
$ python dast_scanner.py http://localhost:8080 --severity HIGH --sarif report.sarif  # CI gate + code scanning
$ python tests/vulnerable_app.py  # start the bundled vulnerable app, then scan http://127.0.0.1:5000
