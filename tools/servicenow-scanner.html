01Overview

ServiceNow is the connective tissue of enterprise IT: it holds identities, roles, workflows, integration credentials, and the change records that run the business. A single misconfigured system property or an over-scoped OAuth client can quietly expose all of it. SSPM-ServiceNow is an open-source SaaS Security Posture Management scanner that audits a live ServiceNow instance for exactly these misconfigurations, without installing an agent, plugin, or scoped app.

The tool talks to the standard ServiceNow Table REST API using read-only Basic Auth credentials, batch-fetches the security-relevant system properties in a single call, then runs 47 distinct checks across 13 posture domains: XSS, CSRF, session management, authentication (lockout, password, Basic Auth, SSO, MFA), password policy, file-upload safety, transport security, server-side script sandboxing, audit logging, user/privileged-access hygiene, OAuth client hygiene, ACL exposure, and email security. Findings are severity-ranked (CRITICAL/HIGH/MEDIUM/LOW), each mapped to a CWE, and rendered to the console, machine-readable JSON, and a self-contained interactive HTML report.

It is deliberately lightweight and safe: one Python file, one external dependency (requests), and read-only queries mean a security engineer can point it at a dev, test, or production instance in minutes and get a prioritized remediation list. Non-zero exit codes on CRITICAL/HIGH findings make it drop-in for CI/CD posture gates.

The check set is grounded in Palo Alto Networks' SSPM for ServiceNow, CIS ServiceNow best practices, and OWASP guidance, making it a practical, auditable posture baseline for SecOps teams, ServiceNow platform admins, and GRC functions who need continuous assurance rather than a once-a-year manual review.

02Key Capabilities

Agentless live REST scanning

Audits a real ServiceNow instance through the standard Table API with read-only Basic Auth credentials, so no plugin, MID Server, or scoped app has to be installed.

Declarative system-property rule engine

28 sys_properties checks are expressed as data with reusable operators (eq_true/eq_false/lte_int/gte_int/not_empty) and a missing_is_fail flag that models ServiceNow's safe defaults.

13-domain posture coverage

Spans XSS, CSRF, sessions, authentication, passwords, file uploads, transport, script sandboxing, audit logging, users, OAuth, ACLs, and email in a single pass.

Identity and privileged-access checks

Flags an active default admin account, stale accounts idle 90+ days, maint-role holders, admins without MFA, service accounts holding admin, and security_admin sprawl.

OAuth integration hygiene

Inspects registered oauth_entity clients for wildcard/admin scope, access tokens living beyond one hour, never-used-but-active clients, and overall client sprawl.

ACL exposure detection

Queries sys_acl for role-less read/write rules on sys_user, core sys_ tables, and log tables that would let any session read or modify sensitive data.

CWE-mapped findings

Every finding carries a CWE reference (e.g. CWE-79, CWE-352, CWE-613, CWE-434, CWE-269) to slot cleanly into existing vulnerability and risk workflows.

Multi-format reporting

Emits a color-coded console report, indented JSON for pipelines, and a dark-themed self-contained HTML report with live severity/category/text filtering.

CI/CD posture gating

Returns exit code 1 whenever any CRITICAL or HIGH finding remains after severity filtering, so it can fail a build or scheduled posture check.

Secret-friendly configuration

Accepts instance, username, and password via CLI flags or SNOW_INSTANCE / SNOW_USERNAME / SNOW_PASSWORD environment variables to keep credentials out of shell history.

Resilient API client

Auto-paginates at 500 records per page and degrades gracefully on 401/403/404/timeout so missing plugins or tables are skipped rather than crashing the scan.

Offline demo mode

run_demo.py injects synthetic findings to produce sample JSON and HTML reports without any live connection, ideal for demos and report-format previews.

03Architecture

A single-file Python scanner organized as a pipeline: a CLI/config layer resolves credentials, a resilient Table-API client batch-fetches the relevant sys_properties and per-table records, a declarative rule engine plus a set of API-driven check modules evaluate them into normalized Finding objects, and a reporting layer renders console, JSON, and interactive HTML output.

1
CLI & configuration
argparse front-end (main) that resolves --instance/--username/--password or SNOW_* env vars, validates required credentials, normalizes short instance names to full service-now.com URLs, and sets severity/output options.
2
REST API client
_api_get performs authenticated GETs against /api/now/table/{table} with automatic 500-record pagination and graceful 401/403/404/timeout handling; _fetch_properties batch-loads all rule properties via a single nameIN query.
3
Declarative property rule engine
PROPERTY_RULES (28 rules) plus _check_system_properties / _evaluate_property apply operator + threshold + missing_is_fail logic against cached sys_properties values.
4
API-driven check modules
Dedicated methods scan live records: _check_users, _check_oauth_apps, _check_acls, _check_audit_logging, _check_email_security, and _check_file_attachments (blocklist diffing against DANGEROUS_EXTENSIONS).
5
Finding model
A single Finding class provides a normalized schema (id, name, category, severity, table, context, description, recommendation, CWE) shared with the author's other scanners for consistent tooling.
6
Reporting layer
summary/filter_severity/print_report/save_json/save_html produce a color console report, indented JSON, and a self-contained dark-themed HTML report with client-side filtering.

05Security Controls

XSS prevention
Verifies glide.ui.escape_text, escape_html_text_area, AntiSamy HTML sanitization, and an active Content Security Policy so user content cannot be rendered as executable HTML (CWE-79/CWE-1021).
CSRF & transport
Checks glide.security.use_csrf_token, HTTPS redirect enforcement (glide.ui.https_redirect), and the SameSite cookie attribute to close cross-site request forgery and cleartext-session vectors (CWE-352/CWE-319/CWE-1275).
Session management
Enforces a <=30-minute idle timeout, bounded active/guest session lifespans, session-ID rotation after login (session fixation), and concurrent-session control (CWE-613/CWE-384).
Authentication & MFA
Flags high login lockout thresholds, blank-password acceptance, Basic-Auth-for-scripted-APIs, unenforced SSO, and disabled platform MFA (CWE-307/CWE-521/CWE-287/CWE-308).
Password policy
Seven checks on minimum length (NIST SP 800-63B), upper/lower/digit/special complexity, application-level lockout count, and password-history depth (CWE-521/CWE-307).
Privileged & user hygiene
Detects an active default admin account, accounts idle 90+ days, maint-role backdoors, admins without MFA enrolled, service accounts holding admin, and security_admin sprawl (CWE-798/CWE-269/CWE-308).
OAuth client hygiene
Reviews oauth_entity clients for admin/wildcard scope, access-token lifetimes over one hour, never-used active clients, and total active-client sprawl (CWE-269/CWE-284/CWE-613).
Access control (ACLs)
Queries sys_acl for role-less read on sys_user, role-less write on core sys_ tables, and unrestricted log-table reads that enable data exposure and tampering (CWE-284/CWE-532).
File upload & email safety
Validates MIME-type checking and the attachment blocklist against a curated dangerous-extension set (exe/bat/ps1/jar/scr...), plus inbound-email antivirus scanning and address filtering (CWE-434/CWE-200).
Audit logging & script sandbox
Confirms platform audit logging, an active syslog/SIEM probe, adequate log retention, server-side script sandboxing, and restricted dynamic-form generation (CWE-778/CWE-250/CWE-20).

07Quick Start

$ pip install requests
$ python servicenow_scanner.py --instance myco --username <read-only-user> --password <pass>
$ python servicenow_scanner.py --instance https://myco.service-now.com -u USER -p PASS --severity HIGH
$ export SNOW_INSTANCE=myco SNOW_USERNAME=user SNOW_PASSWORD=pass && python servicenow_scanner.py
$ python servicenow_scanner.py -i myco -u USER -p PASS --json report.json --html report.html
$ python run_demo.py   # generate sample JSON/HTML reports offline with synthetic findings
