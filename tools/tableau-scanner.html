01Overview

Tableau Cloud SSPM is a read-only SaaS Security Posture Management scanner purpose-built for Tableau Cloud. It connects to a live site over the Tableau REST API using a scoped Personal Access Token, harvests the full configuration surface (users, groups, projects, data sources, workbooks, flows, schedules, and site settings), and evaluates 45 security controls spanning five domains. The result is a severity-weighted posture score from 0 to 100, plus machine-readable JSON and a dark-themed HTML report with per-finding evidence and concrete remediation steps.

BI platforms are a blind spot for most security teams: Tableau Cloud holds curated views of the organization's most sensitive data (finance, HR, PII), yet its configuration drifts as admins, creators, and embedded credentials accumulate. This scanner turns that opaque surface into an auditable posture. It catches the failures that actually lead to exposure -- users bypassing IdP federation, embedded database credentials, unencrypted extracts, over-privileged 'All Users' grants, stale and service accounts, and unlocked default project permissions -- and grades them on a weighted scale so the highest-impact gaps float to the top.

It is designed for cloud security engineers, GRC teams, and Tableau site administrators who need repeatable, evidence-backed access reviews rather than one-off manual audits. Because it is strictly read-only and driven entirely by a PAT, it is safe to run against production and slots naturally into a scheduled GitHub Actions pipeline with score-gating (fail the build if posture drops below a threshold).

The codebase is deliberately small and legible: a three-phase collect -> analyze -> report pipeline, a modular per-domain check architecture built on an abstract base class, 58 unit tests with mock Tableau fixtures, and a CI matrix (Python 3.10-3.12) enforcing lint, bandit, pip-audit, and an 80% coverage gate. No fabricated capabilities -- every control maps to a real REST API field the collector actually pulls.

02Key Capabilities

45 controls across 5 security domains

Evaluates Identity & Authentication, Access Control & Permissions, Data Security, API & Integrations, and Logging & Monitoring -- nine controls each, all grounded in real Tableau REST API fields.

Severity-weighted posture score (0-100)

A single defensible number derived from CRITICAL/HIGH/MEDIUM/LOW weights (25/15/8/3), where PASS earns full weight, WARN earns 50%, and FAIL earns zero -- so noisy low-severity items can't mask a critical gap.

Strictly read-only, PAT-authenticated

Uses a Personal Access Token and only issues GET requests against nine endpoints, so it is safe to run against production without ever modifying the Tableau environment.

Evidence-rich JSON + dark-themed HTML reports

Every finding ships with details, a why-it-matters description, remediation guidance, and a capped list of the exact offending users/resources for immediate triage.

Per-domain category scoring

Breaks the overall score into per-domain percentages (Identity, Access, Data, API, Logging) so teams can see exactly which area is dragging posture down.

CI/CD score gating

The --min-score flag and CRITICAL-failure detection drive the process exit code, letting a pipeline fail automatically when posture regresses below a threshold.

Resilient REST collection

The collector wraps every API call in exponential-backoff retry with configurable connect/read timeouts, tolerating transient Tableau Cloud errors during large-site pagination.

Modular, extensible check suites

Each domain is a suite subclassing a BaseChecks ABC with an _add() helper, so adding a new control is a single method plus a test -- no changes to the orchestrator.

Service account and admin-sprawl detection

Pattern-matches service-account naming, flags admins on local auth, and grades site-role distribution against least-privilege ratios to surface privilege creep.

Containerized and automation-ready

Ships a SHA-pinned, non-root Dockerfile with HEALTHCHECK, a Makefile of dev targets, and a ready-to-use scheduled GitHub Actions workflow.

03Architecture

The scanner is a linear three-phase pipeline -- Collect, Analyze, Score & Report -- wired together by a small CLI. Phase 1 (collector.py) authenticates via PAT and pulls the site's configuration into a single nested dict using the tableauserverclient SDK. Phase 2 (checks/ package) hands that dict to a SecurityChecks orchestrator that delegates to five per-domain suites, each subclassing a BaseChecks ABC and emitting Finding dataclasses. Phase 3 (scoring.py + report.py) computes the severity-weighted score and renders JSON plus a self-contained HTML report.

1
CLI orchestrator (cli.py)
argparse entrypoint with env-var fallback; wires the three phases, configures logging, prints the console summary, and sets the exit code from CRITICAL failures and --min-score.
2
Collection layer (collector.py)
TableauCollector signs in with a PersonalAccessTokenAuth PAT and pulls users, groups, projects, data sources (with connections), workbooks, schedules, flows, and site settings via paginated, retry-wrapped GET calls.
3
Analysis layer (checks/ package)
SecurityChecks.run_all() iterates five suites -- IdentityChecks, AccessControlChecks, DataSecurityChecks, APIChecks, LoggingChecks -- each running nine _check_* methods that record Findings through the BaseChecks._add() helper.
4
Domain model (models.py)
Shared Finding and ScanResult dataclasses plus Severity, Status, and Category enums and the SEVERITY_WEIGHT map that every downstream stage depends on.
5
Scoring engine (scoring.py)
calculate_score() folds findings into total/earned weight buckets to produce the overall 0-100 posture score and a per-category score dict.
6
Reporting layer (report.py)
Serializes the ScanResult to indented JSON and renders a dependency-free dark-themed HTML report via an embedded Jinja2 template.

05Security Controls

Identity & Authentication (AUTH-001..009)
IdP federation (SAML/OIDC) enforcement with break-glass tolerance, stale/never-logged-in account detection (>90 days), site-admin count and duplicate-admin sprawl, auth-method consistency, external/cross-domain users, site-role distribution health, and service-account discovery.
Access Control & Permissions (ACCS-001..009)
Over-privileged Creator accounts, insecure default project permissions, permission-lock (ManagedByOwner) enforcement, guest access, group-based access adoption, dangerous 'All Users' group scope, derived permissions, project-hierarchy depth, and single-owner content concentration.
Data Security (DATA-001..009)
Embedded database credential auditing (CRITICAL), extract encryption at rest, sensitive-data naming (salary/PHI/HIPAA/PII keywords), certification coverage, download surface area, stale sources, Bridge/remote-query-agent usage, multi-connection sources, and uncertified sources in sensitive projects.
API & Integrations (API-001..009)
Dashboard extension allowlisting, revision-history change tracking, subscribe-others and run-now controls, Prep flow security review, content sprawl, stale-flow detection, data-driven alert exposure, and Ask Data / natural-language mode.
Logging & Monitoring (LOG-001..009)
Tableau Catalog (Data Management) enablement, user-visibility controls, commenting/mention controls, access-review readiness, orphaned-content detection, governed access-request workflow, admin-mode configuration, content-ownership distribution, and site activation state.
Severity model
Each control is weighted CRITICAL=25, HIGH=15, MEDIUM=8, LOW=3, INFO=0; PASS earns full weight, WARN 50%, FAIL 0%, SKIP is excluded -- yielding both an overall and per-domain 0-100 score.
Read-only guarantee
Only GET requests are issued across nine REST resources; the collector performs no writes and the PAT requires only read-level Site Administrator scope.

07Quick Start

$ pip install -e .   # install from source (Python 3.10+)
$ export TABLEAU_SERVER=https://your-pod.online.tableau.com TABLEAU_SITE=my-site TABLEAU_TOKEN_NAME=my-pat TABLEAU_TOKEN_SECRET=<secret>
$ tableau-sspm   # runs collect -> analyze -> score, writes JSON + HTML to ./sspm_output
$ tableau-sspm --server <URL> --site <SITE> --token-name <NAME> --token-secret <SECRET> --min-score 65   # fail if posture drops below 65
$ python run_demo.py   # offline demo against mock data, no live Tableau needed
$ docker run --rm -e TABLEAU_SERVER -e TABLEAU_SITE -e TABLEAU_TOKEN_NAME -e TABLEAU_TOKEN_SECRET -v $(pwd)/sspm_output:/app/output tableau-sspm:latest
