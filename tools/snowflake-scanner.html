01Overview

Snowflake has become the crown jewels of the modern data stack — the single place where an organization's customer records, financial data, and analytics converge. That concentration makes account posture a top-tier risk: one over-privileged ACCOUNTADMIN, one password-only login without MFA, or one external stage without a storage integration can expose the entire warehouse. The Snowflake SSPM Scanner is a read-only SaaS Security Posture Management tool that audits a live Snowflake account against the CIS Snowflake Foundations Benchmark v1.0.0 and turns the result into a scored, remediation-ready report.

It ships as a single self-contained Python file. It connects through the official Snowflake Python Connector, runs 39 SQL-based audit checks across four CIS domains — Identity & Access Management, Monitoring & Alerting, Networking, and Data Protection — and produces color-coded console, machine-readable JSON, and a self-contained HTML dashboard. Every finding carries a severity, evidence, concrete remediation, and a four-framework compliance mapping (CIS, NIST 800-53 Rev 5, ISO 27001:2022, SOC 2).

The scanner is deliberately non-invasive: it issues only SHOW commands and SELECTs against SNOWFLAKE.ACCOUNT_USAGE views and never modifies the account. It supports password, RSA key-pair, and externalbrowser SSO authentication, and its severity-weighted scoring engine collapses all 39 findings into a single 0-100 posture score with an A-F letter grade. A configurable --min-score gate and a hard exit code 1 on any CRITICAL failure make it drop-in ready for CI/CD.

It is built for cloud security engineers, data platform owners, and GRC teams who need a fast, auditable, zero-infrastructure way to prove Snowflake compliance and catch drift before an auditor — or an attacker — does.

02Key Capabilities

39 CIS Benchmark checks

Audits a Snowflake account against 39 controls from the CIS Snowflake Foundations Benchmark v1.0.0, spanning identity, monitoring, networking, and data protection.

Read-only by design

Uses only SHOW commands and ACCOUNT_USAGE SELECT queries so it inspects posture without ever altering the account.

Severity-weighted posture score

Collapses all findings into a single 0-100 score with an A-F letter grade using CRITICAL/HIGH/MEDIUM/LOW weights.

Four-framework compliance mapping

Every finding maps to CIS Snowflake v1.0.0, NIST 800-53 Rev 5, ISO 27001:2022, and SOC 2 for instant audit evidence.

Three output formats

Emits ANSI color console output, machine-readable JSON for SIEM/CI, and a self-contained dark-themed HTML dashboard.

Multiple authentication methods

Supports username/password, RSA key-pair (loaded via the cryptography library), and externalbrowser SSO.

CI/CD gating

A --min-score threshold plus exit code 1 on any CRITICAL failure lets pipelines fail builds on posture regressions.

Single-file, one-dependency deployment

All logic lives in one Python file requiring only the Snowflake connector and cryptography, for zero-config portability.

Actionable findings

Each result includes description, evidence, and concrete remediation (often the exact ALTER/SQL command) rather than a bare pass/fail.

Graceful degradation

Checks that cannot be evaluated (missing tags, Enterprise-only features, unverifiable Tri-Secret Secure) return SKIP or WARN and are excluded from scoring instead of failing hard.

Offline demo mode

A bundled run_demo.py generates synthetic reports with no live Snowflake connection so output formats can be validated locally.

03Architecture

A single-file Python scanner organized as a three-phase pipeline: connect, scan, then score-and-report. A SnowflakeClient wraps the Snowflake Python Connector as a read-only SQL executor; a SnowflakeScanner runs 39 CIS check methods that each emit a Finding object; and a scoring plus reporting layer computes a severity-weighted posture score and renders console, JSON, and HTML output.

1
SnowflakeClient (connector layer)
Wraps snowflake-connector-python with connect() for password/key-pair/SSO auth and query()/query_scalar() helpers that return normalized dict rows.
2
SnowflakeScanner (check engine)
Holds the 39 check_X_Y methods grouped by CIS section (1.x IAM, 2.x Monitoring, 3.x Networking, 4.x Data Protection) and run_all() to execute them sequentially.
3
Finding + COMPLIANCE_MAP (data model)
A slotted Finding class carries rule_id, status, severity, evidence, and remediation; COMPLIANCE_MAP joins each rule ID to its CIS/NIST/ISO/SOC2 references.
4
compute_score (scoring engine)
Applies severity weights (CRITICAL 25 to LOW 3), awards full/half/zero credit for PASS/WARN/FAIL, excludes SKIP/ERROR, and derives the 0-100 score and A-F grade.
5
Reporters (output layer)
print_console_report (ANSI), write_json_report (machine-readable), and write_html_report (self-contained dark dashboard) render the findings and score.
6
CLI (build_parser + main)
argparse-based entry point resolving flags and env-var fallbacks, orchestrating the pipeline and setting exit codes for CI gating.

05Security Controls

Identity & Access Management (17 checks, SF-IAM-*)
Verifies SSO and SCIM integration, MFA for password users (CRITICAL), 14+ char password policy, key-pair auth and 180-day rotation for service accounts, inactive-user disablement, admin idle-session timeout, ACCOUNTADMIN/SECURITYADMIN count limits, and that admin roles are not default roles, granted to custom roles, or used to own/run tasks and stored procedures.
Monitoring & Alerting (9 checks, SF-MON-*)
Confirms audit data exists for admin-role grants, MANAGE GRANTS usage, SSO password sign-ins, sign-ins without MFA, security-integration and network-policy changes, SCIM token creation, new share exposures, and unsupported client drivers.
Networking (2 checks, SF-NET-*)
Checks for an account-level network policy and user-level network policies scoped to service accounts to restrict source IPs.
Data Protection (11 checks, SF-DP-*)
Validates periodic rekeying, AES-256 key size, data retention (>=7 days and 90-day critical retention), storage-integration enforcement for stage creation/operations/external stages, PREVENT_UNLOAD_TO_INLINE_URL, Tri-Secret Secure, and data-masking plus row-access policies.
Severity-weighted scoring
CRITICAL=25, HIGH=15, MEDIUM=8, LOW=3, INFO=0; PASS earns full weight, WARN half, FAIL zero, and SKIP/ERROR are excluded, yielding a 0-100 score and A-F grade.
Fail-safe CI gating
Returns exit code 1 when any CRITICAL check FAILs or the posture score falls below the configurable --min-score threshold.
Read-only guarantee
Issues only SHOW and SELECT queries against SNOWFLAKE.ACCOUNT_USAGE views, making the scan non-invasive to production accounts.

07Quick Start

$ pip install -r requirements.txt
$ python snowflake_scanner.py --account xy12345.us-east-1 --user admin_user --password "$SNOWFLAKE_PASSWORD" --role ACCOUNTADMIN --json report.json --html report.html
$ python snowflake_scanner.py --account xy12345.us-east-1 --user admin_user --private-key-path /path/to/rsa_key.p8 --role ACCOUNTADMIN --html report.html
$ python snowflake_scanner.py --account xy12345.us-east-1 --user admin_user --authenticator externalbrowser --role ACCOUNTADMIN
$ python snowflake_scanner.py --json report.json --min-score 70   # gate CI: exit 1 below threshold or on CRITICAL fail
$ python run_demo.py   # generate synthetic reports offline, no Snowflake connection

08Compliance & Frameworks

CIS Snowflake Foundations Benchmark v1.0.0
Primary control source (Oct 2023); all 39 checks map to CIS sections 1.1-4.11 with L1/L2 profile levels.
NIST SP 800-53 Rev 5
Each finding maps to relevant control families (e.g. IA-2 MFA, AC-6 least privilege, AU-6/12 audit, SC-28 encryption).
ISO/IEC 27001:2022
Findings reference Annex A controls such as A.8.5 authentication and A.8.24 cryptography.
SOC 2 Type II
Maps to Trust Services Criteria including CC6 (access) and CC7 (monitoring).
