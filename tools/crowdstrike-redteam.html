01Overview

Before ransomware detonates or a nation-state actor moves laterally, one thing almost always happens first: they blind the defenses. Turning off Windows Defender, bypassing AMSI, unloading Sysmon, killing EDR services, or silencing ETW and the Event Log. MITRE ATT&CK catalogs this as T1562.001 (Impair Defenses: Disable or Modify Tools), and it is among the most frequently observed techniques in real incidents. The open-source Atomic Red Team project documents 59 distinct tests for it. This tool audits your hosts for the fingerprints of every one.

CrowdStrike Red Team Validation is a single-file, zero-dependency Python auditor that reads offline host-configuration exports (services, registry, Defender preferences, Sysmon filter drivers, processes, launchd, sysctl, GuardDuty and more) and checks them against all 59 Atomic tests — organized into 18 categories spanning 6 platforms (Windows, Linux, macOS, AWS, ESXi, O365). It is purely a reader: it executes nothing, changes nothing, and touches no network, so it is safe to run on production or in an air-gapped forensic workflow.

Each hit becomes a structured finding with a severity, the exact tampered setting, a copy-paste remediation command, the ATT&CK technique, and the originating Atomic test GUID — so a blue team can confirm what a red team (or a real adversary) actually disabled. The catalog is explicitly threat-mapped: LockBit Black auto-logon keys, WhisperGate's AdvancedRun, Conti's Set-MpPreference sweep, FIN7's EDR-Freeze, BlackLotus HVCI tampering (CVE-2022-21894), and Awfulshred wiper's history/swap wipes are all represented.

It is built for detection engineers validating EDR coverage, incident responders triaging a suspected compromise, and purple teams turning Atomic Red Team executions into a repeatable verification pass. Because it ships with a sample_data set full of deliberate misconfigurations, you can see it light up 42 findings in one command before pointing it at a real host.

02Key Capabilities

Full T1562.001 Coverage

One auditor method exists for each of the 59 Atomic Red Team tests, so every documented way of disabling or modifying security tools has a corresponding host check.

Six-Platform Reach

Detects impairments on Windows, Linux, macOS, AWS (GuardDuty), ESXi, and O365 from their respective config exports, not just Windows.

Offline & Non-Destructive

Reads only JSON exports and never executes commands, mutates the host, or opens a network connection — safe for production and air-gapped forensic use.

Zero Dependencies

Runs on stock Python 3.8+ using only the standard library, so it drops onto a locked-down responder box with nothing to pip install.

Actionable Findings

Every finding carries a CRITICAL/HIGH/MEDIUM/LOW severity, the exact affected setting, a ready-to-run remediation command, the platform, and the Atomic test GUID.

Threat-Actor Mapping

Checks are annotated to real campaigns — LockBit, WhisperGate, Conti, BlackBasta, Gootloader, FIN7, BlackLotus, Kasseika, Awfulshred, and INC ransomware.

Severity Filtering

A --severity flag narrows the report to CRITICAL/HIGH (or any floor) so responders can triage the loudest signals first.

Bundled Demo Dataset

Ships 19 sample config files seeded with deliberate weaknesses, producing 42 findings out-of-the-box so users can validate the tool before real use.

Defender-Deep Inspection

Covers 13 distinct Windows Defender tampering vectors — Set-MpPreference, registry GPO keys, DISM feature removal, dangerous path/extension/process exclusions, stale signatures, and deleted scheduled tasks.

AMSI, ETW & Sysmon Telemetry Checks

Flags AMSI InitFailed/COM/registry bypasses, disabled ETW Auto Loggers and providers, and unloaded or uninstalled Sysmon drivers that blind script and event visibility.

Structured JSON Report

Emits a machine-readable report with a technique meta-block and per-finding detail, ready to feed a SIEM, ticketing system, or dashboard.

Flexible Export Ingestion

Each config type accepts multiple filename aliases and both list and object JSON shapes, so exports from varied collection scripts load without reformatting.

03Architecture

A single T1562001Auditor class drives a three-stage offline pipeline: it loads a directory of JSON host-configuration exports through an alias-aware file map, runs 59 per-test check methods grouped into 18 lettered categories, and each positive check appends a normalized finding record. The CLI entry point then severity-filters the findings and serializes a JSON report with a technique meta-block. There is no network, database, or external engine — all logic is in one ~969-line stdlib module.

1
Export Loader (load_data)
Maps 26 logical config keys to candidate filenames (e.g. registry.json / registry_export.json), reads each with UTF-8-BOM tolerance, and reports how many of the expected files loaded.
2
Check Engine (run_all_checks)
Invokes all 59 check_test_* methods in category order (A-R); each pulls the relevant export, applies its indicator logic, and emits findings only on a positive match.
3
Category Modules (A-R)
18 thematic groups — Syslog/Linux, macOS, Sysmon, AMSI, Security Services, Defender Tampering, Defender Registry, Office, O365, EDR-Specific, Event Log, LockBit, HVCI, AWS, Defender-on-Linux, ESXi, ASR, ETW — encapsulating each platform's detection routines.
4
Finding Builder (finding)
Normalizes every hit into a dict with test number, check_id, title, severity, category, affected items/count, remediation, platform, Atomic GUID, technique, and timestamp.
5
Report Serializer (main)
Applies the --severity floor, prints a per-category summary table, and writes a JSON report containing a meta-block (technique, total tests, counts, source) plus the findings array.

05Security Controls

Windows Defender Tampering (13 checks)
Detects Set-MpPreference disables (realtime/behavior/script/block-at-first-seen), WinDefend service stop, DisableAntiSpyware registry, DISM/PowerShell feature removal, stale signatures (>14 days), deleted Defender scheduled tasks, and dangerous folder/extension/process exclusions.
AMSI Bypass (4 checks)
Flags amsiInitFailed=True, a deleted AMSI provider registry key, IAmsi COM interface override, and AmsiEnable=0 — the common ways PowerShell script scanning is neutralized.
EDR Kill Tooling (EDR-Specific)
Catches uninstalled CrowdStrike Falcon (Windows/Linux/macOS), Backstab (Kasseika), and EDR-Freeze (FIN7) processes, plus a 14-name security-service disable sweep across MDE/MsSense/major AV vendors.
Telemetry Blinding — ETW, Sysmon & Event Log
Detects disabled ETW Auto Loggers and providers, unloaded/uninstalled Sysmon filter drivers, and a stopped Windows Event Log service (WinPwn/Phant0m).
Linux & macOS Defenses
Checks rsyslog/syslogd, SELinux permissive/disabled, ASLR (randomize_va_space=0), SysRq, swap-off, page-cache drops, cleared shell history, plus macOS Carbon Black, LittleSnitch, OpenDNS, Gatekeeper, and Falcon daemons.
Ransomware TTP Indicators
LockBit Black auto-logon and DisablePrivacyExperience keys, WhisperGate's AdvancedRun, and BlackLotus HVCI tampering (CVE-2022-21894) are individually flagged.
Cloud & Virtualization
Detects AWS GuardDuty suspended/deleted, Microsoft Defender ATP (mdatp) stopped on Linux/macOS, and ESXi account-lockout policy disabled (AccountLockFailures=0).
Defender Hardening Regression
Identifies missing Attack Surface Reduction (ASR) rules and Defender GPO registry keys set to disable protection via reg.exe or PowerShell.
Severity & Remediation
Every finding is graded CRITICAL/HIGH/MEDIUM/LOW and paired with a concrete remediation command (e.g. Set-MpPreference -DisableRealtimeMonitoring 0) plus the Atomic test GUID for traceability.

07Quick Start

$ git clone https://github.com/Krishcalin/CrowdStrike-Red-Teaming-Test.git && cd CrowdStrike-Red-Teaming-Test
$ python t1562_001_audit.py --data-dir ./sample_data --output report.json
$ python t1562_001_audit.py --data-dir ./my_exports --severity HIGH
$ Export host config first (Windows): Get-MpPreference | ConvertTo-Json -Depth 3 | Out-File .\host_exports\defender_preferences.json (see README for the full per-platform export scripts)
$ Review report.json — each finding includes test_number, severity, category, affected_items, remediation, and atomic_guid
