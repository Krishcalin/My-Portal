01Overview

Palo Alto Network Security Scanner is an offline configuration auditor for Palo Alto Networks NGFW (PAN-OS) and Prisma SASE deployments. Instead of touching production firewalls over an API, it ingests the config exports teams already have — running-config XML from a PA-Series, VM-Series, CN-Series, or Panorama box, plus JSON/CSV exports from Strata Cloud Manager and Prisma Access — and grades them against the CIS Palo Alto Benchmarks, PAN-OS best practices, and IronSkillet baselines. Nothing leaves the analyst's machine and no device connectivity is required, which makes it safe to run in change-control windows, air-gapped assessment environments, and third-party audits.

What sets it apart is dual coverage in one tool. Most PAN-OS hardening scripts stop at the on-prem firewall; this one carries the same policy-as-data discipline into the SASE world, with a dedicated module for ZTNA 2.0 enforcement, Secure Web Gateway, CASB, DLP, Prisma SD-WAN, and cloud-identity MFA. The result is a single scan that spans the classic security stack (App-ID policy hygiene, threat prevention, SSL decryption, GlobalProtect, HA) and the cloud-delivered edge.

The engine is deliberately boring in the best way: pure Python 3.8+ standard library, zero third-party dependencies, and a small BaseAuditor framework that every module extends. Each of the 68 checks emits a structured finding with severity, affected items, GUI remediation path, and a benchmark or best-practice reference, and the run produces an interactive HTML dashboard with a weighted risk score and severity/category breakdowns. A separate, clearly-fenced red-team module ships an authorized-use validator for the 2024 PAN-OS management-interface auth-bypass chain (CVE-2024-0012 / CVE-2024-9474).

It is aimed at firewall engineers, cloud-security teams, and consultants who need a repeatable, evidence-producing review of Palo Alto estates — the kind of artifact that drops straight into a hardening ticket or an audit workpaper.

02Key Capabilities

Dual NGFW + SASE coverage

One scan audits both on-prem PAN-OS firewalls and Prisma Access / Prisma SD-WAN SASE fabric, including a dedicated ZTNA 2.0 / CASB / DLP module.

Fully offline analysis

Reads exported config files only — no API tokens, no device connectivity, safe to run in air-gapped or change-controlled environments.

Zero dependencies

Runs on Python 3.8+ standard library alone (xml.etree, json, csv, re) so it installs and executes anywhere without a package manager.

Ten modular auditors

Device, policy, threat, network, decryption, GlobalProtect, logging, HA, SASE, and URL/data modules can be run individually or all at once.

Multi-format config parser

A unified PanConfig loader normalizes PAN-OS XML, Prisma/SCM JSON, and CSV exports into one queryable model with XPath and JSON helpers.

CIS Benchmark alignment

Checks map to CIS PAN-OS Benchmark sections 1.x-9.x, with PAN-OS Best Practices and Hardening Guide references cited in flagged findings; IronSkillet baselines inform check design at the project level.

Interactive HTML dashboard

Generates a self-contained report with a weighted risk score, severity and category breakdowns, and client-side filtering and search.

Severity and module filtering

CLI flags let you scope a run to specific modules or a minimum severity (CRITICAL/HIGH/MEDIUM/LOW) for targeted reviews.

Custom baseline overrides

A --config JSON lets teams tune thresholds to their own standard instead of the shipped defaults.

Actionable remediation

Each flagged finding carries an exact PAN-OS GUI navigation path and a benchmark citation so fixes can be applied without guesswork.

Authorized red-team validator

A fenced, non-destructive validator confirms exposure to CVE-2024-0012 / CVE-2024-9474 by version fingerprint and auth-bypass probe, with JSON evidence output.

03Architecture

A thin CLI orchestrator (pan_scanner.py) loads all config exports from a directory into a single PanConfig object, then dispatches it through ten domain auditors that each extend a shared BaseAuditor. Every auditor emits normalized finding dicts, which are aggregated, severity-filtered, and rendered by a standalone ReportGenerator into an interactive HTML dashboard. A separate redteam package holds an active-scan CVE validator that is intentionally decoupled from the offline pipeline.

1
CLI orchestrator (pan_scanner.py)
Parses arguments, maps module keys to auditor classes, runs selected modules, applies severity filtering, tallies findings, and triggers report generation.
2
Config loader / PanConfig parser (base.py)
load_all_configs walks a directory, ingests XML/JSON/CSV via a FILE_MAP, detects device type (ngfw/panorama/prisma_access), and exposes XPath, regex, and JSON/CSV query helpers.
3
BaseAuditor framework
Defines the severity constants and the finding() factory that stamps each result with check_id, category, affected items, remediation, references, and timestamp.
4
Ten domain auditor modules
device_mgmt, security_policy, threat_prevention, network_security, decryption, globalprotect, logging_fwd, ha_resilience, sase_security, and url_data_filter each implement run_all_checks over the parsed config.
5
Findings model
A uniform list of structured finding dicts flows from every module, decorated with device hostname and type before aggregation.
6
ReportGenerator (report_generator.py)
Computes a weighted risk score and severity/category rollups, then renders a self-contained interactive HTML dashboard with filtering.
7
Red-team validation module (redteam/)
A standalone urllib/ssl-based validator for CVE-2024-0012 / CVE-2024-9474 with connectivity, version, auth-bypass, and session-creation probes and JSON reporting.

05Security Controls

Device & Management (10 checks)
Password complexity, admin lockout thresholds, management permitted-IP restrictions, idle timeout, login banner, SNMPv2c detection, minimum TLS version, superadmin sprawl, default admin account, and API key lifetime.
Security Policy (8 checks)
Flags any/any and overly permissive source rules, missing App-ID, unlogged rules, rules without security profiles, absent intrazone-deny, disabled rules, and undocumented rules.
Threat Prevention (8 checks)
Verifies antivirus, anti-spyware, and vulnerability-protection profiles with blocking actions, WildFire forwarding, file blocking, DNS Security, and scheduled dynamic content updates.
SASE / Prisma Access (9 checks)
Audits Prisma Access status, ZTNA 2.0 continuous trust and per-app access, Secure Web Gateway, CASB and shadow-IT detection, DLP data patterns/actions, SD-WAN encryption, cloud-identity MFA, remote-network IPsec, and ADEM.
Decryption & SSL/TLS (6 checks)
SSL decryption policy coverage, forward-proxy configuration, decryption profiles, TLS protocol settings, and certificate expiry.
GlobalProtect & VPN (6 checks)
Gateway/portal authentication, HIP enforcement, split-tunnel exposure, certificate validation, and IPsec crypto strength.
Network Security (6 checks)
Zone protection profiles, zone coverage, interface management profiles, DoS protection, and packet-buffer protection.
Logging & HA (10 checks)
External/syslog forwarding, log-forwarding profiles, forwarded log types, log encryption, and config-change logging; HA configuration, passive link state, preemption, link/path monitoring, and HA2 encryption.
URL & Data Filtering (5 checks)
URL filtering profile presence, high-risk category handling, credential-phishing prevention, safe search enforcement, and data filtering.
Red-team CVE validation
Non-destructive detection of CVE-2024-0012 (CVSS 9.8 auth bypass via X-PAN-AUTHCHECK) and CVE-2024-9474 (CVSS 6.9 privilege escalation) via version fingerprinting and header/session probes, gated behind an explicit --confirm authorization flag.

07Quick Start

$ git clone https://github.com/Krishcalin/PaloAlto-Network-Security.git && cd PaloAlto-Network-Security
$ python pan_scanner.py --data-dir ./sample_configs --output report.html
$ python pan_scanner.py --data-dir ./exports --modules policy threat sase
$ python pan_scanner.py --data-dir ./exports --severity HIGH
$ python pan_scanner.py --data-dir ./exports --config custom_baseline.json
$ python redteam/cve_2024_0012_validator.py --target https://firewall.internal --confirm  # authorized use only
