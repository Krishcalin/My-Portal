01Overview

Windows CIS Hardening & Scanners is a dependency-free PowerShell toolkit that both *applies* and *audits* CIS Benchmark security baselines across the modern Windows estate. It pairs an active hardening script for Windows 11 Enterprise with three read-only compliance scanners for Windows Server 2016, 2019, and 2022 - so the same team can enforce a baseline on endpoints and continuously verify it on servers using one consistent, self-contained toolset.

The Windows 11 script (CIS_Win11_Hardening.ps1) applies 170+ settings from the CIS Microsoft Windows 11 Enterprise Benchmark v4.0.0 across ten categories - account and lockout policy, UAC/NTLM/SMB/LSA security options, service disablement, the three firewall profiles, advanced audit policy, computer and user administrative templates, TLS/cryptography, legacy network protocol shutdown (LLMNR, NetBIOS, mDNS), and Windows Defender Attack Surface Reduction rules. It is idempotent, backs up the full security state before any change, supports Level 1 and Level 2 profiles, offers a -WhatIf dry run, and ships as a single file ready to drop into an AD GPO Computer Startup Script.

The three server scanners are strictly read-only - safe to run on production - and audit 318 (2016), 360 (2019), and 392 (2022) controls, 1,070 in total. They share a data-driven engine: every control is a structured object (CIS ID, profile, severity, check type, expected value, operator, recommendation, and CWE) evaluated by a generic comparison engine. They auto-detect server role, translate well-known SIDs to friendly names for User Rights checks, and emit colored console output, machine-readable JSON, and a self-contained interactive HTML report - returning a CI/CD-friendly non-zero exit code when CRITICAL or HIGH controls fail.

The toolkit is built for Windows platform engineers, security operations, and compliance teams who need auditor-defensible CIS coverage without agents, cloud consoles, or third-party modules. With 1,240+ total controls and settings expressed in ~6,000 lines of native PowerShell, it is easy to review, easy to GPO-deploy, and honest about its blast radius - the scanners never touch the system, and the hardening script clearly flags the Level 2 changes that can break RDP, Bluetooth, and printing.

02Key Capabilities

Active Windows 11 CIS Hardening

Applies 170+ settings from the CIS Windows 11 Enterprise Benchmark v4.0.0 across ten security categories in a single self-contained script.

Three Read-Only Server Scanners

Audits Windows Server 2016, 2019, and 2022 against their respective CIS Benchmarks - 1,070 controls total - without ever modifying system state.

Level 1 / Level 2 Profiles

Both hardening and scanning support CIS L1 (broadly compatible) and L2 (high-security) profiles, with Member Server and Domain Controller variants for servers.

Data-Driven Control Engine

Each control is a structured object evaluated by a generic comparison engine supporting EQ, RANGE, SUBSET, IN, MATCH, EMPTY and more, so coverage grows by adding data, not code.

Idempotent Hardening with Full Backup

The Win11 script skips already-compliant settings and exports secedit, auditpol, firewall, and registry state before any change so every run is reversible and repeatable.

WhatIf Dry-Run & Backup-Only Modes

Preview every proposed change without touching the system, or export the current baseline for review before committing to a rollout.

Server Role Auto-Detection

Scanners read Win32_ComputerSystem.DomainRole to classify Domain Controller, Member Server, or Standalone and apply only the relevant CIS controls.

User Rights Assignment with SID Translation

Resolves 30+ well-known SIDs from secedit exports to friendly names so privilege-assignment checks are readable and accurate.

Interactive HTML & JSON Reporting

Scanners emit a self-contained dark-theme HTML report with filtering plus machine-readable JSON carrying CIS ID, status, expected vs actual, CWE, and recommendation.

CWE-Mapped Severity Scoring

Every control carries a CRITICAL/HIGH/MEDIUM/LOW severity and a CWE reference, and scanners compute a color-coded compliance percentage.

GPO-Ready Deployment

The hardening script is a single file with no external dependencies, designed to run as an Active Directory Computer Startup Script with an optional Windows 11 WMI filter.

CI/CD-Friendly Exit Codes

Scanners return exit code 1 when any CRITICAL or HIGH control fails and 0 otherwise, so compliance can gate a pipeline.

03Architecture

The toolkit is four independent, self-contained PowerShell scripts sharing common design patterns rather than a shared library. The Win11 hardening script is an imperative sequence of Set-* section functions that read current state, compare, and apply via native tools (secedit, registry, Set-Service, netsh, auditpol, Set-MpPreference). The three server scanners share a data-driven architecture: CIS controls are declared as arrays of PSCustomObjects, dispatched by check type to collectors that read live state, evaluated by a generic Compare-Value operator engine, accumulated as results, and rendered to console, JSON, and HTML.

1
Control Definitions (data layer)
Per-section arrays of PSCustomObjects carrying CIS_ID, Title, Profile, Category, Severity, CheckType, CheckKey, Operator, Expected, Recommendation, and CWE - the single source of truth for coverage.
2
State Collectors
Type-specific readers - Get-RegistryValue, Get-SeceditExport/Value, Get-AuditpolSetting, Get-ServiceStartType, plus firewall and User Rights readers - that pull live configuration from the OS.
3
Comparison Engine (Compare-Value)
A generic evaluator implementing EQ/NE/GE/LE/GT/LT/RANGE/MATCH/IN operators (plus CONTAINS/NOTMATCH/NOTIN/NOTEXIST); the parallel User Rights evaluator (Test-UserRightControl) adds SUBSET/EMPTY/EXACT logic, together deciding PASS/FAIL/WARN for each control independent of check type.
4
Profile & Role Resolver
Test-ProfileApplicable and Get-ServerRole gate which controls run based on the selected L1/L2/MS/DC/BL/NG profile and the detected Domain Controller / Member Server role.
5
Result Accumulator
Add-ScanResult and Invoke-Controls collect outcomes into an ArrayList and maintain Passed/Failed/Warning/Error counters and the compliance score.
6
Reporting Layer
Show-ConsoleReport, Export-JsonReport, and Export-HtmlReport render colored console output, machine-readable JSON, and a self-contained interactive HTML report; exit code is derived from CRITICAL/HIGH failures.
7
Hardening Apply Layer (Win11)
Set-RegistryValue, Set-AuditSubcategory, and the Set-* section functions perform idempotent, WhatIf-aware changes after Backup-CurrentSettings snapshots the baseline.

05Security Controls

Account & Lockout Policies
Password history (>=24), max/min age, length (>=14), complexity, reversible-encryption disabled, lockout duration/threshold/reset, and Administrator account lockout - enforced via secedit and evaluated by the scanners.
User Rights Assignment
46 privilege-assignment checks in the 2022 scanner (SeDebugPrivilege, SeTcbPrivilege, SeImpersonatePrivilege, deny-logon rights, etc.) with SUBSET/EMPTY logic and well-known SID-to-name translation.
Security Options (2.3)
UAC, NTLM/LAN Manager auth level, SMB client/server signing (CRITICAL), LSA protection, anonymous access restriction, Microsoft-account blocking, and interactive-logon banners and inactivity limits.
System Services Hardening
Disables risky services - 24 at L1, extended to 37 at L2 adding Bluetooth, RDP, Print Spooler, WinRM, and SMB Server on Windows 11; scanners verify service start types.
Windows Defender Firewall
Enforces and audits Domain, Private, and Public profiles (~30 firewall settings on Win11) via registry and netsh, including logging and default inbound/outbound actions.
Advanced Audit Policy
Configures and checks ~29-33 audit subcategories via auditpol (logon/logoff, account management, privilege use, object access, policy change, system) to guarantee forensic visibility.
Administrative Templates
~80 computer-scope and ~10 user-scope registry policies covering PowerShell logging, LAPS, credential/lateral-movement mitigations, autoplay, and telemetry controls.
Attack Surface Reduction & Defender
Applies 13 + 7 Defender ASR rules via Set-MpPreference and registry on Win11; server scanners check Tamper Protection, Controlled Folder Access, and ransomware/WMI-persistence ASR rules.
TLS/Cryptography & Legacy Protocols
Disables TLS 1.0/1.1 and weak ciphers (24 registry settings) and shuts down LLMNR, NetBIOS-over-TCP, and mDNS name-resolution poisoning vectors (8 settings).
Next Generation & BitLocker Security
Scanner NG/BL profiles check Virtualization-Based Security, Credential Guard, HVCI, System Guard Secure Launch, Kernel DMA Protection, and BitLocker drive encryption.

07Quick Start

$ .\CIS_Win11_Hardening.ps1 -Profile L1   # Apply Level 1 hardening (recommended baseline)
$ .\CIS_Win11_Hardening.ps1 -Profile L1 -WhatIf   # Dry run - show changes without modifying the system
$ .\CIS_Win11_Hardening.ps1 -BackupOnly   # Export current security state for review
$ .\CIS_WinServer2022_Scanner.ps1 -ScanProfile L1_DC   # Scan as a Level 1 Domain Controller
$ .\CIS_WinServer2022_Scanner.ps1 -ScanProfile All -JsonPath scan.json -HtmlPath report.html   # Full scan with JSON + HTML export
$ .\CIS_WinServer2022_Scanner.ps1 -ScanProfile L1_MS -MinSeverity HIGH   # Show only HIGH and CRITICAL findings
