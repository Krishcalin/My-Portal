01Overview

Alpha-AI is an agentic offensive-security framework that turns an LLM into a hands-on operator. It exposes a curated set of Kali penetration-testing binaries to a Claude/GPT/Copilot agent over two parallel surfaces — a FastMCP server (stdio transport, for Claude Desktop/Code) and a FastAPI REST API — so the same tool wrapper is written once and reachable from either. The agent can then chain reconnaissance → vulnerability discovery → exploitation → reporting against explicitly authorized targets, with every invocation normalized into structured Finding objects.

The design principle that makes it interesting to a security leader is that it is authorization-first. A YAML whitelist (literal hosts, CIDR ranges, and glob patterns) is enforced by a central dispatcher before any remote tool touches a host; a target that isn't listed raises UnauthorizedTargetError, which the REST layer maps to HTTP 403. Only local-only lookups such as searchsploit opt out. A single tool registry is the source of truth for both surfaces, a disk cache keyed on sha256(tool, target, args) avoids re-running expensive scans, and structlog records tool name, target, argv, return code, and duration for every run.

This is deliberately an early-stage MVP (v0.1.0). It ships 9 tool wrappers across five categories — nmap, nuclei, gobuster, ffuf, sqlmap, enum4linux, crackmapexec, hydra, and searchsploit — each backed by a pure, unit-tested parser that converts raw tool output into severity-rated findings. It is built for red-teamers and AI-security researchers who want to experiment with autonomous, tool-using pentest agents inside a controlled Kali container, with a documented pattern for adding tool #10 in five files. Roadmap phases target broader tool breadth, an LLM autopilot loop, Markdown/HTML reporting, and MITRE ATT&CK mapping.

02Key Capabilities

Dual MCP + REST exposure

The same tool set is reachable both as MCP tools (FastMCP, stdio) for Claude Desktop/Code and as JSON REST endpoints (FastAPI on :8000), written once and shared.

Authorization-first execution

A YAML target whitelist supporting literals, CIDRs, and globs blocks any tool from running against a host that hasn't been explicitly authorized.

Single tool registry

Every wrapper self-registers with one ToolRegistry, so MCP shims and REST routes build their surfaces from the same source of truth with no duplicated logic.

Central dispatcher

One Dispatcher enforces authorization, cache lookup, tool invocation, and cache write for both surfaces, so security policy lives in exactly one place.

Result caching

A disk-backed JSON cache keyed on sha256(tool, target, args) skips re-running expensive scans, with a use_cache=False override to force a fresh run.

Async subprocess runner

CommandRunner executes Kali binaries asynchronously with per-tool timeouts, PATH validation via which(), timeout-kill handling, and structured exec/done logging.

Pure, testable parsers

Each tool has an I/O-free parse_* function that turns raw XML/JSON/text into normalized Finding objects with severities, evidence, references, and CVEs.

Severity-rated findings

Parsers apply per-tool severity policy — sqlmap/hydra findings are always CRITICAL, crackmapexec Pwn3d! is CRITICAL, gobuster sensitive paths MEDIUM — for consistent triage.

Structured audit logging

structlog records tool name, target, argv, return code, and duration for every invocation, giving a machine-readable trail of agent activity.

Kali container runtime

A Dockerfile on the kalilinux/kali-rolling base preinstalls the tool binaries and ships alpha-mcp / alpha-api entrypoints for reproducible deployment.

Documented extension pattern

CLAUDE.md specifies the exact five-file recipe (parser, wrapper, registry import, MCP shim, REST route, plus a parser test) to add a new tool.

Threat-intel data models

An included threat_intel module defines IP/domain/file-reputation and MITRE ATT&CK mapping models (currently mock-backed) as scaffolding for future enrichment.

03Architecture

Alpha-AI is a layered Python framework. An LLM agent calls in over one of two surfaces — FastMCP (stdio) or FastAPI (HTTP) — both of which delegate to a shared Dispatcher. The dispatcher consults the tool registry, enforces the target-authorization whitelist, checks the result cache, then invokes the tool wrapper, which builds an argv, runs it through the async CommandRunner, and hands stdout to a pure parser that emits normalized, severity-rated Finding objects packaged in a ToolResult.

1
Exposure surfaces (servers/)
mcp_server.py (FastMCP @mcp.tool shims) and rest_api.py (FastAPI POST /tools/<name> routes) present identical capabilities to LLM agents.
2
Dispatcher
servers/dispatcher.py centralizes the request flow: registry lookup → authorization gate → cache get → tool fn → cache put → ToolResult.
3
Tool registry
core/registry.py holds ToolSpec metadata (name, category, description, parameters, requires_authorization) and load_builtin_tools() imports the 9 wrappers so they self-register.
4
Core services
core/auth.py (TargetAuthorizer whitelist), core/cache.py (sha256 JSON cache), core/runner.py (async subprocess + timeout), and core/logging.py (structlog) provide cross-cutting infrastructure.
5
Tool wrappers (tools/)
Thin per-tool modules under recon/web/network/cred/exploit that build argv, call the runner, and delegate parsing — no business logic.
6
Parsers (parsers/)
Pure functions (parse_nmap_xml, parse_nuclei_jsonl, parse_sqlmap, …) that convert raw tool output into list[Finding] with severity, evidence, references, and CVEs.
7
Models (core/models.py)
Pydantic v2 types — Severity, ToolCategory, Target, Finding, CommandResult, ToolResult — that normalize output across every tool.
8
Threat intel (threat_intel/)
Standalone reputation and MITRE ATT&CK mapping models with mock lookup sources; scaffolding not yet wired into the MCP/REST surfaces.

05Security Controls

Target authorization
TargetAuthorizer loads a YAML whitelist and matches literals, CIDR networks, and fnmatch globs; require() raises UnauthorizedTargetError (a PermissionError subclass) that the REST API maps to HTTP 403. Enforced in the dispatcher before any remote tool runs.
Recon — nmap
Port scanning and service detection (-Pn -T<timing> -p <ports> -sV -oX -); XML parsed to open-port findings, one INFO finding per open port.
Web — nuclei
Template-based vulnerability scanning; JSONL parsed with nuclei severity mapped 1:1 to Alpha-AI Severity and classification.cve-id captured onto findings.
Web — gobuster / ffuf
Directory and content brute-forcing; discovered paths rated by policy — sensitive paths (admin, backup, .git, .env, config, wp-admin, phpmyadmin) flagged MEDIUM, other 200s LOW.
Web — sqlmap
Automated SQL-injection detection in batch mode with level/risk/technique controls; any confirmed injection point is emitted as CRITICAL.
Network — enum4linux
SMB/Samba enumeration of shares, users, and OS banner; disk shares MEDIUM, users LOW, OS INFO.
Network — crackmapexec
Multi-protocol (smb/…) network and AD checks; a Pwn3d! success line is CRITICAL, other [+] successes HIGH.
Credential access — hydra
Login brute-forcing across SSH/FTP/SMB/RDP/HTTP/MySQL/Postgres with stop-on-first support; any valid credential found is CRITICAL.
Exploit — searchsploit
Local Exploit-DB lookup (requires_authorization=False, the only opt-out); severity by exploit type — remote/webapps HIGH, local/dos MEDIUM — with exploit-db.com references.
Execution safety & audit
Async CommandRunner validates binaries with which() (raising ToolNotFoundError), enforces per-tool timeouts with kill-on-timeout, never raises on non-zero exit, and logs argv/return code/duration via structlog for every run.

07Quick Start

$ docker build -t alpha-ai .   # build the Kali container with all tool binaries
$ docker run -it --rm alpha-ai alpha-mcp   # run the MCP server over stdio for Claude Desktop/Code
$ docker run -it --rm -p 8000:8000 alpha-ai alpha-api   # run the REST API on :8000
$ curl -X POST http://localhost:8000/tools/nmap -H 'content-type: application/json' -d '{"target":"127.0.0.1","ports":"22,80,443","timing":4}'
$ curl http://localhost:8000/tools   # list all registered tools (also GET /health)
$ python -m pytest tests/test_parsers.py tests/test_auth.py tests/test_registry.py tests/test_dispatcher.py -q   # run the parser/auth/registry tests on a non-Linux host

08Compliance & Frameworks

MITRE ATT&CK
threat_intel module maps behaviors to ATT&CK techniques (e.g. T1071.001, T1003.001, T1021.002); roadmap Phase 4 targets per-finding technique mapping and Navigator layer export.
CVE
nuclei findings capture classification.cve-id and the Finding model carries a cve list, so vulnerabilities are tracked by CVE identifier.
Exploit-DB (EDB)
searchsploit findings reference exploit-db.com entries by EDB-ID for known-exploit correlation.
