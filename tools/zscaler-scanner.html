01Overview

Enterprises betting on Zscaler's Zero Trust Exchange collapse their perimeter into a cloud SASE fabric — but a single permissive URL rule, an SSL-inspection bypass, or an over-broad ZPA access policy quietly reopens the door attackers walk through. The Zscaler Security Scanner is an agentless, offline posture-assessment tool that ingests exported ZIA, ZPA, and DSPM configuration and flags the policy gaps, weak defaults, and least-privilege violations that drift into production over time.

The scanner runs 62 discrete checks across 12 audit modules spanning three platforms: ZIA secure web gateway (URL filtering, SSL/TLS inspection, cloud firewall, threat prevention/sandbox), ZPA Zero Trust access (access-policy least privilege, application segmentation, App Connector health), and the surrounding control plane (DLP, Cloud App Control, DSPM, admin/MFA hardening, IdP/SAML, and audit-log forwarding). Each finding is ranked CRITICAL to LOW and ships with remediation guidance, then rolled up into a self-contained HTML dashboard.

Because it works purely on JSON exports and has zero external dependencies (Python 3.8+ standard library only), it needs no live API credentials or network path to the tenant — making it safe to run in change-review, audit, and air-gapped assessment workflows. Security engineers, cloud/network architects, and auditors can point it at a config snapshot and get an evidence-backed picture of how far a deployment has drifted from a Zero Trust baseline.

It is designed for teams operating Zscaler at scale who need repeatable, reviewable posture assessments rather than one-off console spot-checks — the modular design lets you scan a single control area (say, ZPA access policy or SSL inspection) or the entire estate in one pass.

02Key Capabilities

Agentless offline assessment

Analyzes exported ZIA/ZPA/DSPM JSON configuration with no live tenant credentials or network access, so it runs safely in audit and air-gapped review workflows.

ZIA secure web gateway audit

Inspects URL filtering, SSL/TLS inspection, cloud firewall, and threat-prevention policies for permissive defaults and inspection blind spots.

ZPA Zero Trust access review

Validates private-access policy for least privilege, device-posture checks, user-risk scoring, and default-deny enforcement across application segments.

Application segmentation & connector health

Flags wildcard app segments, oversized domain scopes, bypass-enabled segments, and unhealthy App Connectors that widen the private-access attack surface.

DLP and DSPM data-protection checks

Detects monitor-only DLP rules, missing dictionaries/EDM, and absent data classification or exposure monitoring in Data Security Posture Management.

Cloud App Control & Shadow IT

Surfaces risky sanctioned apps, disabled SaaS discovery, and missing tenant restrictions that let unmanaged cloud usage bypass policy.

Admin, MFA & IdP hardening

Catches unenforced admin MFA, excess super-admins, dormant accounts, stale API keys, missing IdP/SCIM, and weak SAML configuration.

Audit-log & compliance integrity

Verifies audit logging, NSS streaming, retention thresholds, and detects forwarding rules that bypass Zscaler inspection entirely.

Severity-ranked, remediation-rich findings

Every check emits a structured finding with CRITICAL–LOW severity, affected items, remediation steps, and references for fast triage.

Modular targeted scanning

Run the full estate or a single control area via the --modules flag (e.g. zpa-policy, zia-ssl) plus severity filtering for focused reviews.

Self-contained HTML dashboard

Aggregates results into a shareable HTML report with a summary rollup, falling back to structured JSON when needed.

Zero-dependency Python

Runs on Python 3.8+ standard library alone — no pip installs, no SDKs — making deployment and portability trivial.

03Architecture

The scanner is a staged pipeline: a CLI orchestrator selects modules and loads a directory of exported JSON configs through a DataLoader, hands them to twelve auditor classes built on a shared BaseAuditor/Finding framework, then aggregates severity-ranked findings into an HTML (or JSON) report. All logic is pure Python standard library with no live API calls.

1
CLI & orchestration (zscaler_scanner.py)
argparse entry point that parses --data-dir, --modules, --severity, --output and --config, then dispatches the selected auditors and applies severity filtering.
2
Data ingestion (DataLoader)
Loads exported ZIA/ZPA/DSPM JSON files from the data directory, tolerating multiple filename variants and reporting load statistics.
3
Auditor framework (base.py — BaseAuditor / Finding)
Shared base class defining SEVERITY_CRITICAL/HIGH/MEDIUM/LOW and a finding() factory producing structured records with check ID, category, remediation, references and timestamp.
4
Audit modules (zia_security, zia_zpa_modules, platform_dspm)
Twelve auditor classes across three files implement run_all_checks() for URL/SSL/firewall/threat, DLP/app-control/ZPA-policy/ZPA-app, and admin/auth/DSPM/audit domains.
5
Reporting (report_generator.py — ReportGenerator)
Renders aggregated findings into a self-contained HTML dashboard with summary statistics, falling back to JSON output if report generation is unavailable.

05Security Controls

ZIA URL Filtering (6 checks)
Flags unblocked high-risk categories (CRITICAL), allowed uncategorized URLs, disabled caution pages, missing safe-search enforcement, absent rules, and a permissive default allow-all rule.
SSL/TLS Inspection (5 checks)
Detects no inspection policy (CRITICAL), risky bypass rules, disabled certificate-pinning detection, a minimum TLS version set too low, and inspection gaps for Client Connector traffic.
Cloud Firewall (5 checks)
Checks for missing firewall rules, a default rule that is not deny/block, absent DNS control rules, rules without logging, and large numbers of disabled rules.
Threat Prevention & Sandbox (6 checks)
Verifies Cloud Sandbox is configured and enabled, malicious-URL blocking is on (CRITICAL), advanced threat protection exists, and vulnerable-browser and file-type controls are active.
DLP & Data Protection (4 checks)
Identifies missing DLP policy, monitor-only (non-blocking) rules, absent standard dictionaries, and no Exact Data Match.
Cloud App Control & Shadow IT (4 checks)
Surfaces missing app-control policy, risky applications allowed, disabled SaaS/Shadow-IT discovery, and unconfigured tenant restrictions.
ZPA Access Policy — ZTNA least privilege (6 checks)
Catches no access-policy rules (CRITICAL), overly permissive rules, missing device-posture checks, no user-risk scoring, a default rule that is not deny, and excessive re-auth timeouts.
ZPA Application Security & Connector Health (5 checks)
Flags wildcard application segments, unhealthy App Connectors, segments spanning too many domains, bypass-enabled segments, and segments missing server groups.
Admin, Authentication & IdP (9 checks)
Enforces admin MFA (CRITICAL), limits super-admins and dormant accounts, requires admin-IP restrictions and API-key rotation, and validates IdP presence, SCIM provisioning, SAML strength, and surrogate-IP timeouts.
DSPM & Audit/Logging/Compliance (12 checks)
Confirms DSPM, data classification, and exposure monitoring are active, and validates audit logging, NSS streaming, 365-day retention, authenticated locations, and no Zscaler-bypassing forwarding rules.

07Quick Start

$ git clone https://github.com/Krishcalin/Zscaler-Security-Scanner && cd Zscaler-Security-Scanner
$ python zscaler_scanner.py --data-dir ./sample_data --output zscaler_security_report.html
$ python zscaler_scanner.py --data-dir ./exports --modules zia-ssl zia-url zpa-policy
$ python zscaler_scanner.py --data-dir ./exports --severity HIGH
$ python zscaler_scanner.py --data-dir ./exports --modules zpa-policy zpa-app  # ZPA Zero Trust review only
$ python zscaler_scanner.py --data-dir ./exports --config baseline.json --modules all

08Compliance & Frameworks

Zero Trust Network Access (ZTNA)
Built around Zscaler's Zero Trust Exchange model — ZPA modules enforce least-privilege access, device posture, user-risk scoring, and default-deny.
SASE secure-access baseline
Audits the ZIA secure web gateway and ZPA private-access controls that constitute a SASE deployment's security posture.
Data protection (DLP / DSPM)
Checks data-loss prevention enforcement and data classification/exposure monitoring, aligning with data-governance control expectations.
Audit & log-retention controls
Validates audit logging, NSS streaming, and a 365-day retention threshold consistent with common logging-and-monitoring control requirements.
