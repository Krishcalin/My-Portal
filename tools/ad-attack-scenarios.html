01Overview

Active Directory is the single richest target in most enterprise networks, and the paths to Domain Admin are well known: Kerberoasting, AS-REP roasting, delegation abuse, AD CS certificate-template misconfigurations, DCSync, and stale-credential exposure. This tool turns that adversary playbook into a repeatable, read-only assessment. It connects to a Domain Controller over LDAP/LDAPS (and optionally WinRM), enumerates the exact objects and settings an attacker would abuse, and reports where the environment is exposed — without ever writing to the directory.

Under the hood it runs a modular battery of checks organised into 15 categories — domain configuration, Kerberos, password policy, privileged accounts, group policy, LDAP, DNS, trusts, AD Certificate Services (ESC1–ESC8), replication/DCSync, object permissions, service accounts, authentication hardening, lateral movement, and audit logging. Every finding ships with severity, the observed vs. expected state, a plain-language rationale, and a ready-to-paste PowerShell remediation command, so a defender can close the gap immediately.

Results roll up into a 0–100 posture score with an A–F letter grade, a self-contained interactive HTML report (dark theme, severity filtering and search), and machine-readable JSON. Severity-based exit codes (0 clean / 1 HIGH / 2 CRITICAL) make it drop-in for CI/CD gating. It is built for red teamers mapping an AD attack surface, blue teams hardening it, and consultants who need a fast, safe, evidence-backed domain review.

It is deliberately safe-by-design: assessment is performed exclusively through LDAP searches and PowerShell Get-* queries, with WinRM-dependent checks degrading gracefully to INFO notices when WinRM is unavailable. A Domain Users account gives roughly 70% coverage via LDAP; adding registry and audit-policy checks over WinRM brings the rest.

02Key Capabilities

80 checks across 15 attack categories

Audits the full AD attack surface — Kerberos, AD CS, privileged accounts, delegation, trusts, replication and more — from a single command.

Read-only, safe-by-design

Uses only LDAP searches and PowerShell Get-* queries, so it never modifies the directory and is safe to run against production DCs.

AD CS ESC1–ESC8 coverage

Detects certificate-template and CA misconfigurations (SAN+client-auth, Any Purpose EKU, EDITF_ATTRIBUTESUBJECTALTNAME2, HTTP enrollment/NTLM relay) that lead to domain takeover.

Kerberos attack-path detection

Flags Kerberoastable and AS-REP roastable accounts, unconstrained/constrained/RBCD delegation, DES-only encryption, aged krbtgt passwords, and an empty Protected Users group.

PowerShell remediation for every finding

Each result carries a copy-paste Set-AD*/Add-ADGroupMember remediation command so defenders can fix issues without hunting for the fix.

Posture score with letter grade

Aggregates findings into a 0–100 posture score and an A–F grade for an at-a-glance measure of domain security health.

Interactive HTML + JSON reports

Produces a self-contained dark-theme HTML report with severity filtering and search, plus machine-readable JSON for pipelines.

CI/CD-ready exit codes

Returns 0 (clean), 1 (HIGH) or 2 (CRITICAL) so scans can gate builds or scheduled compliance jobs.

Dual data sources with graceful degradation

Combines LDAP/LDAPS object data with optional WinRM registry/audit checks, automatically skipping WinRM checks with INFO notices when unavailable.

Selectable scans and YAML profiles

Run individual categories via --checks or drive coverage from a reusable YAML scan profile for consistent, repeatable assessments.

03Architecture

A single CLI orchestrator wires together two read-only collectors (LDAP/LDAPS and optional WinRM), a registry of 15 pluggable checker modules, a severity/scoring engine, and a report generator. Data flows one way: the CLI selects categories (flags or YAML profile), connects to the DC, hands the shared clients to each *Checker class whose run_all() emits structured findings, which are then aggregated, scored into a 0–100 posture grade, and rendered to HTML/JSON with a severity-based process exit code.

1
CLI orchestrator (ad_security_scanner.py)
Parses arguments, loads the YAML profile, resolves enabled categories from ALL_CHECKERS, drives connection setup, runs checkers, prints the summary, and sets the severity-based exit code.
2
LDAP collector (utils/ldap_client.py)
ADLdapClient performs read-only LDAP/LDAPS binds (simple or NTLM) and object/group searches, providing ~70% of coverage for domain, Kerberos, privilege and AD CS data.
3
WinRM collector (utils/winrm_client.py)
ADWinRMClient runs PowerShell Get-* commands for registry and audit-policy checks; optional and degrades gracefully when unreachable.
4
Check modules (checks/, 15 modules)
Each category is a *Checker class with a run_all() method that executes discrete _check_ routines and appends structured findings (id, severity, actual/expected, recommendation, remediation_cmd).
5
Scoring engine (utils/severity.py)
Computes the 0–100 posture score, maps it to an A–F grade, and tallies severity counts used by the summary and exit codes.
6
Report generator (utils/report_generator.py)
Renders the aggregated findings into a self-contained interactive HTML report and a JSON export with run metadata.

05Security Controls

Kerberos Security (8 checks)
Kerberoastable SPN accounts (admins flagged CRITICAL), AS-REP roasting (DONT_REQUIRE_PREAUTH), unconstrained/constrained/RBCD delegation, DES-only encryption, krbtgt password age (Golden Ticket exposure), and empty Protected Users group.
AD Certificate Services (ESC1–ESC8)
ESC1 templates allowing SAN + client auth, ESC2 Any-Purpose/no-EKU templates, ESC6 EDITF_ATTRIBUTESUBJECTALTNAME2 on the CA, and ESC8 HTTP web enrollment vulnerable to NTLM relay.
Privileged Accounts
Domain/Enterprise/Schema Admin membership, stale admin accounts, orphaned adminCount, Kerberoastable admins, and membership of dangerous built-in groups.
Replication / DCSync
Detects principals holding DCSync (directory-replication) permissions, plus SYSVOL/DFSR and replication-health issues.
Password Policy
Minimum length (CIS 14+ benchmark), complexity, lockout thresholds, history, reversible encryption, Fine-Grained Password Policies, and stale passwords.
Authentication Hardening
Legacy and downgrade risks: NTLMv1, LM hash storage, WDigest credential caching, SMB signing, SMBv1, and NTLM restriction.
LDAP Security
LDAP signing enforcement, channel binding, anonymous bind exposure, and LDAPS availability.
Trusts & Object Permissions
SID filtering and selective authentication on trusts, plus Pre-Windows-2000 compatible group and AdminSDHolder tampering on domain ACLs.
Lateral Movement & Service Accounts
LAPS deployment, RDP restrictions, Restricted Admin / Remote Credential Guard, and gMSA adoption vs. privileged/stale service accounts.
Group Policy, DNS, Domain Config & Audit Logging
Audit policy, PowerShell/command-line logging, Credential Guard and unlinked GPOs; DNS zone transfers/dynamic updates/WPAD; functional level, AD Recycle Bin and machine-account quota; and event-log sizing.

07Quick Start

$ git clone https://github.com/Krishcalin/Active-Directory-Security-Assessment.git && cd Active-Directory-Security-Assessment
$ pip install -r requirements.txt
$ python ad_security_scanner.py --server dc01.corp.com --domain corp.com --user scanner@corp.com --password MyP@ss --html report.html --json report.json
$ python ad_security_scanner.py --server dc01.corp.com --domain corp.com --user scanner@corp.com --password MyP@ss --ssl --no-winrm
$ python ad_security_scanner.py --server dc01.corp.com --domain corp.com --user scanner@corp.com --password MyP@ss --checks kerberos_security,password_policy
$ python -m unittest tests.test_scanner -v
