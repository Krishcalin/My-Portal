01Overview

Linux CIS Hardening is a pair of production-grade Bash scripts that lock down Red Hat Enterprise Linux 8 and 9 (plus AlmaLinux, Rocky Linux, and CentOS Stream derivatives) against the official CIS Benchmarks — RHEL 8 v3.0.0 and RHEL 9 v2.0.0. Instead of a 100-page checklist and a weekend of manual edits, an operator runs one script as root and the host is systematically hardened across the kernel, network stack, SSH, authentication, auditd, logging, cron, firewall, and file-integrity layers.

The scripts are built for real fleets, not lab demos. Every change is idempotent (re-runnable without drift), each config file is backed up with a timestamp before it is touched, SSH is validated with `sshd -t` before the daemon is ever restarted to prevent remote lockout, and package operations tolerate already-absent packages so a run never aborts halfway. RHEL 9 goes well beyond RHEL 8, layering in a modern defense-in-depth stack: AIDE file-integrity monitoring, fapolicyd application allowlisting, USBGuard device control, a default-deny nftables firewall, and dnf-automatic security patching.

Beyond remediation, each script finishes with an informational audit phase that inventories world-writable files, SUID/SGID binaries, empty passwords, duplicate UIDs/GIDs, orphaned home directories, and dangerous dotfiles (`.rhosts`, `.netrc`, `.forward`) — written to `/tmp/<hostname>_audit/` for review rather than auto-remediated, so nothing surprising happens to a running production box.

It targets platform engineers, SREs, and security teams who need repeatable, auditable, CIS-aligned baselines for RHEL estates. It is fully transparent: two readable, dependency-free shell scripts plus a 524-line README that documents every control and maps it back to its CIS section number.

02Key Capabilities

Dual RHEL 8 & RHEL 9 coverage

Separate purpose-built scripts track CIS RHEL 8 Benchmark v3.0.0 and CIS RHEL 9 Benchmark v2.0.0, each tuned to the distro's native tooling (yum vs dnf, direct sshd_config vs drop-in).

Lockout-safe SSH hardening

SSH config is validated with `sshd -t` before the daemon is restarted, so a bad directive never strands you out of a remote box.

Idempotent, re-runnable design

Shared helper functions update-or-append config keys in place, so the scripts can run repeatedly without duplicating lines or causing config drift.

Timestamped backups before every change

sshd_config, auditd.conf, sysctl, chrony, nftables and more are copied to a per-host audit directory with a timestamp, giving a clear rollback trail.

Comprehensive auditd rule sets

Ships 51 (RHEL 8) and 58 (RHEL 9) audit rules covering time changes, identity files, logins, permission/ownership changes, mounts, deletions, privileged commands, and kernel-module operations.

Deep kernel & network sysctl hardening

Applies 32–35 sysctl parameters — full ASLR, kptr/dmesg restrictions, ptrace scoping, reverse-path filtering, SYN-cookie protection, redirect/source-route rejection, and IPv6 disable.

RHEL 9 defense-in-depth stack

Installs and configures AIDE integrity monitoring, fapolicyd allowlisting, USBGuard, a default-deny nftables firewall, and dnf-automatic security-only patching in a single pass.

Non-destructive audit reporting

A final phase inventories world-writable files, SUID/SGID binaries, duplicate UIDs/GIDs, and risky dotfiles as findings for manual review — deliberately not auto-remediated.

Strong cryptographic posture

Enforces the system-wide FUTURE crypto policy plus curated SSH Ciphers, ETM-preferred MACs, and modern Curve25519/large-group KexAlgorithms.

Attack-surface reduction

Removes legacy servers (rsh, telnet, tftp, ypserv), compilers, and unneeded daemons, and blacklists legacy filesystem and network-protocol kernel modules.

Pre-flight guardrails

Each script enforces root privileges and gates on the detected OS version before making any change, refusing to run on an unsupported release.

Operator-aware post-run guidance

On completion the scripts print explicit next steps — replace the banner company name, tune nftables ports, review container namespace settings, and reboot to activate module/sysctl/crypto changes.

03Architecture

The tool is not an application but two self-contained, dependency-free Bash scripts (RHEL8 Hardening Script at 776 lines, RHEL9 Hardening Script at 941 lines) run directly on the target host as root. Each follows the same pipeline: a strict-mode preamble with pre-flight guardrails, a set of shared idempotent config-editing helpers, a sequence of numbered remediation sections that write to canonical system paths, then a backup trail and an informational audit-reporting stage. There is no build step or external runtime — the scripts orchestrate native RHEL tooling (sysctl, augenrules, authselect, update-crypto-policies, nmcli, nft, dnf/yum).

1
Pre-flight guardrails
`set -euo pipefail`, root/EUID check, and `/etc/os-release` version gate that abort the run on unsupported releases before any change is made.
2
Idempotent config helpers
Reusable functions (`_set_conf_option`/`_set_kv`, `_set_ini`, `_set_sshd`) that update-or-append keys in key=value, INI, and sshd_config files without creating duplicates.
3
Remediation sections
26 (RHEL 8) and 35 (RHEL 9) numbered blocks that write modprobe blacklists, remove/install packages, disable services, and configure PAM, SSH, sysctl, cron, and the firewall.
4
Backup & rollback trail
Every mutated file (sshd, auditd, sysctl, chrony, nftables, PAM) is copied to `/tmp/<hostname>_audit/` with a date-time stamp before modification.
5
Defense-in-depth services (RHEL 9)
Initialises AIDE, enables fapolicyd and USBGuard, loads a default-deny nftables ruleset, and schedules dnf-automatic security updates.
6
Informational audit reporting
Final read-only stage that scans filesystems and account databases, writing findings logs (SUID/SGID, world-writable, duplicate UIDs, risky dotfiles) without remediating them.

05Security Controls

Kernel & memory hardening
Full ASLR (randomize_va_space=2), kptr_restrict=2, dmesg_restrict, yama ptrace_scope (1 on 8, 2 on 9), perf_event_paranoid=3, sysrq off, suid_dumpable=0, protected hardlinks/symlinks (plus fifos/regular=2 and user.max_user_namespaces=0 on RHEL 9).
Network stack hardening
Reverse-path filtering, TCP SYN cookies, disabled IP forwarding, ICMP redirect/source-route rejection, martian-packet logging, broadcast-ICMP ignore, and full IPv6 disable via sysctl and modprobe.
SSH hardening (CIS 5.2)
PermitRootLogin/EmptyPasswords/UserEnvironment/HostbasedAuth disabled, MaxAuthTries 4, LoginGrace 60, client-alive timeouts, all forwarding off, VERBOSE logging, and curated Ciphers/MACs(ETM)/KexAlgorithms — validated with sshd -t before restart.
Audit logging (CIS 4.1)
auditd enabled with 8192 buffer and panic-on-loss failure mode (-f 2); 51/58 rules track time-change, identity files, logins/faillock, perm/ownership, unsuccessful access, mounts, deletions, privileged commands, module load/unload, and SELinux MAC policy; RHEL 9 adds ENRICHED format and finit_module.
Authentication & PAM
pwquality minlen=14 with credit/repeat rules, pam_faillock (deny=5, unlock_time=900 on RHEL 9), su restricted to wheel via pam_wheel, login.defs password aging (90/7/7) and SHA512 with 5000 rounds on RHEL 9.
Firewall (RHEL 9, CIS 3.5)
Default-deny stateful nftables ruleset (drop policy, loopback allow, invalid-packet drop, established/related accept, rate-limited ICMP, SSH allow, log-and-drop) with firewalld/iptables disabled to avoid conflicts.
File integrity & allowlisting (RHEL 9)
AIDE database initialised with a daily cron integrity check emailed to root, fapolicyd application allowlisting enabled, and USBGuard policy generated from connected devices.
Attack-surface reduction
Removes legacy services (rsh, telnet-server, tftp, ypserv, talk, xinetd), compilers, and unneeded daemons (bind, samba, squid, net-snmp, postfix); blacklists cramfs/freevxfs/jffs2/hfs/squashfs/udf/usb-storage and dccp/sctp/rds/tipc modules; disables wireless via nmcli.
Logging & time integrity
rsyslog auth/kern/daemon routing with 0640 file mode, persistent compressed journald forwarded to syslog, and (RHEL 9) hardened chrony NTP bound to localhost running as the unprivileged chrony user.
System integrity auditing
Read-only reporting of world-writable and un-owned/un-grouped files, SUID/SGID binaries, empty passwords, duplicate UIDs/GIDs/names, missing groups, orphaned home dirs, root PATH integrity, and .rhosts/.netrc/.forward presence.

07Quick Start

$ sudo bash "RHEL8 Hardening Script"   # harden a RHEL 8 / CentOS Stream 8 / Alma 8 / Rocky 8 host
$ sudo bash "RHEL9 Hardening Script"   # harden a RHEL 9 / Alma 9 / Rocky 9 host
$ Review backups and findings under /tmp/<hostname>_audit/ after the run
$ Replace YOUR_COMPANY_NAME in /etc/issue.net and /etc/motd, and review /etc/nftables/main.nft to open only required ports (RHEL 9)
$ Reboot the host to apply kernel-module, sysctl, crypto-policy, and IPv6 changes
