# PhalanxCyber — Tool Infographic Briefs

Drop one PNG per tool into `assets/infographics/` named exactly `<tool-id>.png` (filenames below).
The tool page auto-embeds it on the right of the hero; tools without an image use the single-column hero.

## Shared style guide (keep all 54 cohesive)

- **Canvas:** landscape ~1400 x 780 px (≈16:9). PNG.
- **Background:** very light, airy (near-white / #F3F7FB) — matches the light-default UX.
- **Palette:** brand cyan/teal (#06B6D4 / #0891B2) for data & flow, violet (#7C3AED) accent, deep-navy (#0F172A) text; reserve **orange** only for threat/alert elements.
- **Style:** semi-illustrative modern SaaS — flowing data-stream lines, circular icon nodes with short labels, a central engine/hub, subtle telemetry/binary texture. Airy, not cluttered.
- **Header:** tool name as a display title + a 3-4 word tagline (as in the LogOcean piece).
- **Text:** a few clear labels only; let the diagram carry the story.

### `app-security-scanner.png` — Static Application Security Testing  ·  _Application Security_
- **Depict:** Each scanner is a self-contained single Python file that follows one identical pipeline: module-level rule dictionaries feed a path walker and file dispatcher, which routes each file to a per-line regex SAST scan and a manifest version-range CVE analyzer; matches become Finding objects (memory-efficient via __slots__) that pass through a severity filter into console/JSON/HTML reporting with exit-code gating. The Microsoft 365 SSPM scanner reuses the same Finding/report layers but swaps the file-walker front-end for a live Microsoft Graph API collector authenticated with OAuth2 client credentials.
