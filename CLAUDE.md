# CLAUDE.md — Phalanx Cyber Portal

## Project Overview

Static web portal showcasing 44 open-source security tools across 9 domains. No backend — HTML + CSS + vanilla JS. Deployed via GitHub Pages.

## Repository Structure

```
index.html                     # Main portal with tool catalogue
about.html                     # About / founder page
strategy.html                  # Cybersecurity strategy page
css/styles.css                 # All styles (dark theme, glassmorphic cards, responsive grid)
js/tools.js                    # TOOLS array (44 entries), TOOL_CATEGORIES, TOOL_ICONS
js/main.js                     # Card rendering, search, filters, detail expansion, counter animations
js/translations.js             # i18n (EN, NL, DE) for hero, nav, stats, footer
assets/favicon.svg             # Favicon
generate_reports.py            # Batch generator: scanner JSON/HTML → portal report pages
generate_overviews.py          # Batch generator: tool data → overview pages
synthetic_data/                # JSON data for tools without local repos (Ariba, Zscaler)
*-report.html (35 files)       # Self-contained sample report pages
*-overview.html (9 files)      # Self-contained tool overview pages
LICENSE                        # GPL-3.0
```

## Architecture

### Tool Data (`js/tools.js`)

Each tool in the `TOOLS` array has:
```
id, name, tagline, category, tags[], stats{}, version, language,
github, status, icon, highlights[],
sampleReport? (string — report HTML filename),
overview? (string — overview HTML filename)
```

**Categories:** `appsec`, `cloud`, `saas`, `infra`, `threat`, `redteam`, `easm`, `risk`

**Icons:** `code`, `globe`, `api`, `sap`, `brain`, `cloud`, `shield`, `saas`, `firewall`, `container`, `terminal`, `factory`, `target`, `radar`

### Card Rendering (`js/main.js`)

- `renderCards()` generates `<article class="tool-card">` from TOOLS array
- Three action buttons per card:
  - **GitHub** — always shown (primary cyan button)
  - **Sample Report** — shown if `tool.sampleReport` exists (ghost button, cyan border)
  - **Overview** — shown if `tool.overview` exists (ghost button, purple border)
- `expandToolDetail(toolId)` toggles inline detail panel below card
- Search filters by tool name, tagline, tags, and language

### Report Pages

Generated by `generate_reports.py`. Reads scanner output (JSON or HTML), extracts findings, and produces self-contained HTML with:
- Severity dashboard (clickable cards)
- Posture score ring gauge: `Score = 100 - (CRIT*15 + HIGH*5 + MED*2 + LOW*0.5)`
- Category bar chart
- Expandable finding accordions
- Filter controls (severity, category, search)
- Compliance section

Supports multiple JSON formats: standard findings array, CRQ scenarios, module_results (Windows Red Team), device vulnerabilities (OT), and bare arrays.

For HTML source files, parses `<div class="finding-card">` elements from scanner-generated reports.

### Overview Pages

Generated by `generate_overviews.py`. Creates capability showcase pages for non-scanner tools with stats grid, feature highlights, category tags, architecture summary, and compliance mapping.

## Key Conventions

- **Tool count:** Currently 44 — update in `index.html` (5 places), `js/translations.js` (6 places: EN/NL/DE badge + subtitle)
- **Stats counter:** `data-target` attribute on `.stats__number` spans drives the animation
- **Design tokens:** All in `:root` in `css/styles.css` — `--bg`, `--accent-1` (#00D4FF), `--accent-2` (#7B61FF)
- **Fonts:** Inter (400–900) + JetBrains Mono (report pages only)
- **All report/overview pages are self-contained** — inline CSS, inline JS, embedded data. No external dependencies except Google Fonts.

## Development Guidelines

- Keep `js/tools.js` as the single source of truth for tool data
- Report/overview pages should be regenerated via the Python scripts, not hand-edited
- When adding tools: update TOOLS array, tool count in index.html + translations, and run generators
- Tool cards are rendered dynamically from JS — there's no static HTML for individual cards
- The `TOOL_CATEGORIES` array auto-counts from `TOOLS.filter()` — no manual count needed
- Translations must stay in sync across EN, NL, DE for hero badge and subtitle

## Running Generators

```bash
# Regenerate all 35 sample report pages
python generate_reports.py

# Regenerate all 9 overview pages
python generate_overviews.py
```

Both read from their internal `TOOLS_CONFIG` / `TOOLS` arrays and write HTML to the portal root.
